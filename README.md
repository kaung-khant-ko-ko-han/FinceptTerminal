## အကြောင်းအရာ

**Fincept Terminal v4** သည် သန့်စင်သော native C++20 desktop application တစ်ခုဖြစ်သည်။ ၎င်းသည် UI နှင့် rendering အတွက် **Qt6** ကိုသုံးပြီး၊ ပိုင်းခြားစိတ်ဖြာမှုအတွက် **Python** ကို ထည့်သွင်းထားကာ၊ professional terminal-class performance ကို native binary တစ်ခုတည်းဖြင့် ပေးအပ်သည်။

### နည်းလမ်း ၂ — အမြန်စတင်ခြင်း (တစ်ချက်နှိပ် Build)

Repository ကို clone လုပ်ပြီး setup script ကို ဖွင့်ပါ — ၎င်းက dependencies အားလုံးကို အလိုအလျောက် တပ်ဆင်ပေးပြီး app ကို build လုပ်ပေးသည်:

```bash
# Linux / macOS
git clone https://github.com/Fincept-Corporation/FinceptTerminal.git
cd FinceptTerminal
chmod +x setup.sh && ./setup.sh
```

Script က အောက်ပါတို့ကို လုပ်ဆောင်သည်: compiler စစ်ဆေးခြင်း, CMake, Qt6, Python, build, နှင့် launch ။

> **Windows:** setup script မရှိပါ — အောက်ပါ နည်းလမ်း ၄ ရှိ manual build အဆင့်များကို သုံးပါ။ ၎င်းသည် command နှစ်ခုသာဖြစ်သည်။

### နည်းလမ်း ၄ — ရင်းမြစ်မှ Build လုပ်ခြင်း (Manual)

> **ဗားရှင်းများကို သတ်မှတ်ထားသည်။** အောက်ပါ ဗားရှင်းအတိအကျကိုသာ သုံးပါ။ အသစ် သို့မဟုတ် အဟောင်း ဗားရှင်းများသည် အထောက်အပံ့မရှိဘဲ build မအောင်မြင်နိုင် သို့မဟုတ် မတည်ငြိမ်သော binary များကို ထုတ်ပေးနိုင်သည်။

#### လိုအပ်ချက်များ (ဗားရှင်းအတိအကျ)

| Tool | သတ်မှတ်ဗားရှင်း | မှတ်ချက် |
|------|----------------|-------|
| **Git** | နောက်ဆုံးထွက် | — |
| **CMake** | **3.27.7** | [ဒေါင်းလုဒ်](https://cmake.org/download/) |
| **Ninja** | **1.11.1** | [ဒေါင်းလုဒ်](https://github.com/ninja-build/ninja/releases) |
| **C++ compiler** | **MSVC 19.38** (VS 2022 17.8) / **GCC 12.3** / **Apple Clang 15.0** (Xcode 15.2) | C++20 လိုအပ်သည် |
| **Qt** | **6.8.3** | [Qt Online Installer](https://www.qt.io/download-qt-installer) |
| **Python** | **3.11.9** | [python.org](https://www.python.org/downloads/release/python-3119/) |
| **Platform SDK** | Win10 SDK 10.0.22621.0 / macOS SDK 14.0 (deploy 11.0+) / glibc 2.31+ | — |

#### Qt 6.8.3 တပ်ဆင်ခြင်း

**Windows:** Qt Online Installer → `Qt 6.8.3 > MSVC 2022 64-bit` ကိုရွေးပါ (တပ်ဆင်လမ်းကြောင်း: `C:/Qt/6.8.3/msvc2022_64`)

**Linux:** Qt Online Installer → `Qt 6.8.3 > Desktop gcc 64-bit` (တပ်ဆင်လမ်းကြောင်း: `~/Qt/6.8.3/gcc_64`). **သို့မဟုတ်** system packages အတွက် `qt6-base-dev qt6-charts-dev qt6-tools-dev qt6-base-private-dev libqt6websockets6-dev libgl1-mesa-dev` တပ်ဆင်ပါ — သတိပြုရန်: system packages များသည် 6.x မိုင်နာ ကွဲပြားနိုင်သည်။

**macOS:** Qt Online Installer → `Qt 6.8.3 > macOS` (တပ်ဆင်လမ်းကြောင်း: `~/Qt/6.8.3/macos`)

#### Build (CMake presets သုံးခြင်း — အကြံပြုသည်)

```bash
git clone https://github.com/Fincept-Corporation/FinceptTerminal.git
cd FinceptTerminal/fincept-qt
```

**အဆင့် ၁ — Configure** (တစ်ကြိမ်သာ၊ သို့မဟုတ် `CMakeLists.txt` ပြောင်းလဲပြီးနောက်):
```powershell
cmake --preset win-release      # Windows (PowerShell)
cmake --preset linux-release    # Linux
cmake --preset macos-release    # macOS
```

**အဆင့် ၂ — Compile** (code အပြောင်းအလဲတိုင်းအတွက် ဤ command ကိုဖွင့်ပါ):
```powershell
cmake --build --preset win-release      # Windows
cmake --build --preset linux-release    # Linux
cmake --build --preset macos-release    # macOS
```

> **စက်ဟောင်း သို့မဟုတ် RAM အကန့်အသတ်ရှိသူများအတွက်:** `--parallel 4` (သို့မဟုတ် ဂဏန်းသေးသေးတစ်ခု) ထည့်ပါ။ ပုံမှန်အားဖြင့် core အားလုံးကို ပြည့်ဝစွာသုံးသည်၊ ၎င်းသည် စက်အဟောင်းများကို အပူလွန်စေပြီး သင့်စနစ်၏ကျန်အပိုင်းကို နှေးစေနိုင်သည်။ ဥပမာ: `cmake --build --preset macos-release --parallel 4`.

Debug variants: `release` ကို `debug` ဖြင့် အစားထိုးပါ (ဥပမာ `win-debug`, `linux-debug`, `macos-debug`).

> **Windows prerequisite:** `~/Documents/PowerShell/Microsoft.PowerShell_profile.ps1` ရှိ PowerShell profile သည် terminal အသစ်တိုင်းတွင် VS 2022 ကို auto-initialize လုပ်သည် — PowerShell အသစ်ဖွင့်ပြီး cmake ကို တိုက်ရိုက်သုံးနိုင်သည်။

#### Build (manual — presets များက သင်၏ Qt path ကို ရှာမတွေ့ပါက)

```powershell
# Windows (PowerShell)
cmake -B build/win-release -G Ninja -DCMAKE_BUILD_TYPE=Release `
  -DCMAKE_PREFIX_PATH="C:/Qt/6.8.3/msvc2022_64"
cmake --build build/win-release
```

```bash
# Linux
cmake -B build/linux-release -G Ninja -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_PREFIX_PATH="$HOME/Qt/6.8.3/gcc_64"
cmake --build build/linux-release

# macOS
cmake -B build/macos-release -G Ninja -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_OSX_DEPLOYMENT_TARGET=11.0 \
  -DCMAKE_PREFIX_PATH="$HOME/Qt/6.8.3/macos"
cmake --build build/macos-release
```

#### Run

```bash
./build/<preset>/FinceptTerminal         # Linux / macOS (preset build)
.\build\<preset>\FinceptTerminal.exe     # Windows (preset build)
```

#### ပြဿနာဖြေရှင်းခြင်း

1. **"Could not find Qt6 6.8.3"** — `CMAKE_PREFIX_PATH` သည် Qt 6.8.3 တပ်ဆင်မှုကို ညွှန်ပြကြောင်း စစ်ဆေးပါ၊ 6.5/6.6/6.8 မဟုတ်ရ။
2. **MSVC ဗားရှင်းအမှား** — VS 2022 17.8+ (MSVC 19.38+) သုံးပါ။ `cl /?` ဖြင့် စစ်ဆေးပါ။
3. **ကွဲပြားသော Qt မိုင်နာဖြင့် အလုပ်လုပ်ရန် လိုအပ်လျှင်** `-DFINCEPT_ALLOW_QT_DRIFT=ON` ထည့်ပါ (local testing အတွက်သာ — releases သို့မဟုတ် CI အတွက် ဘယ်တော့မှမသုံးရ။)
4. Clean rebuild: `build/<preset>/` ကိုဖျက်ပြီး configure ပြန်လုပ်ပါ။

**Fincept Terminal** သည် သမားရိုးကျ software များ၏ ကန့်သတ်ချက်များကို လက်မခံသူများအတွက် တည်ဆောက်ထားသော open-source ငွေကြေးပလက်ဖောင်းတစ်ခုဖြစ်သည်။ ကျွန်ုပ်တို့သည် **ပိုင်းခြားစိတ်ဖြာမှု အနက်နှင့် ဒေတာရယူနိုင်မှု** တို့ဖြင့် ယှဉ်ပြိုင်သည် — အတွင်းသတင်း သို့မဟုတ် သီးသန့် feed များပေါ်တွင် မဟုတ်ပါ။

မကြာသေးမီက build များသည် **Data Sources → Alternative Data** တွင် optional **Adanos Market Sentiment** connectivity ကို အထောက်အပံ့ပေးသည်။ သတ်မှတ်ပါက Equity Research သည် Reddit, X, finance news, နှင့် Polymarket တို့မှ cross-source retail sentiment snapshot များကို ပြသနိုင်သည်။ Adanos ချိတ်ဆက်မှု အသက်မဝင်ပါက အဆိုပါအင်္ဂါရပ်သည် မလှုပ်ရှားဘဲ ကျန် app သည် ယခင်အတိုင်း ပြုမူသည်။

- **Native performance** — C++20 နှင့် Qt6, Electron/web overhead မရှိ
- **Single binary** — Node.js, browser runtime, JavaScript bundler မရှိ
- **Full buy-side analyst toolkit** — equity, portfolio, derivatives, fixed income, corporate finance, alternatives
- **100+ data connectors** — Yahoo Finance မှ government databases အထိ
- **Free & Open Source** (AGPL-3.0) နှင့် commercial licenses ရရှိနိုင်
