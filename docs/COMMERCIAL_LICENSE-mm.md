- **"Modified Version"** ဆိုသည်မှာ module၊ file၊ function၊ screen၊ API၊ data integration၊ service endpoint၊ သို့မဟုတ် analytical algorithm တစ်ခုခုကို ထပ်ဆောင်း၊ ဖယ်ရှား၊ အစားထိုး၊ ပြန်လည်ချိတ်ဆက်၊ ပိတ်၊ အစားထိုး၊ ဘာသာပြန်၊ သယ်ဆောင် (port)၊ သို့မဟုတ် ပြန်လည်ပုံသွင်းခြင်း အပါအဝင် ပြောင်းလဲထားသော Software ၏ မည်သည့်ဗားရှင်းကိုမဆို ခေါ်သည်။

ထည့်သွင်းထားသော၊ API၊ RPC, message queue, file drop, screen scrape, သို့မဟုတ် မည်သည့် ကြားခံမှတစ်ဆင့် ခေါ်ဆိုသော၊ သို့မဟုတ် သိသိသာသာ ပြန်လည်ထုတ်လုပ်သော မည်သည့် software၊ အလုပ်ထွက်ကုန်၊ သို့မဟုတ် စနစ်ကိုမဆို၊ အရင်းအမြစ် သို့မဟုတ် object ပုံစံဖြင့် ခေါ်သည်။ Derivative Work တွင် အောက်ပါတို့ အကန့်အသတ်မရှိ ပါဝင်သည်- (i) Software ၏ source tree ကို ပြုပြင်မွမ်းမံမှုဖြင့်ဖြစ်စေ၊ မရှိဘဲဖြစ်စေ စုစည်းခြင်း (compilation) (ii) Software ၏ အခြားအစိတ်အပိုင်းကို ထိန်းသိမ်းထားစဉ် module၊ API၊ data integration၊ သို့မဟုတ် service endpoint တစ်ခုခုကို အစားထိုး၊ ဖယ်ရှား၊ ပိတ်၊ သို့မဟုတ် ပြန်လည်ချိတ်ဆက်ခြင်း (iii) ၎င်း၏ ဗိသုကာ၊ screen အပြင်အဆင်များ၊ အလုပ်အသွယ်သွယ်၊ sequence-structure-and-organization၊ MCP tool taxonomy၊ broker abstraction layer၊ agent framework အဆင့်ဆင့်၊ terminal command syntax၊ function-code conventions၊ သို့မဟုတ် အင်္ဂါရပ်အစုကို ထိန်းသိမ်းထားချိန်တွင် Software ၏ အစိတ်အပိုင်းတစ်ခုခုကို အခြားသော ပရိုဂရမ်းမင်းဘာသာစကား သို့မဟုတ် runtime သို့ ဘာသာပြန်၊ သယ်ဆောင်၊ သို့မဟုတ် ပြန်လည်ရေးသားခြင်း (iv) UI စာသား၊ terminal commands၊ သို့မဟုတ် function codes များကို အခြားဘာသာစကား သို့မဟုတ် အက္ခရာသို့ ဘာသာပြန်ခြင်း (v) ၎င်း၏ အဓိက ရည်ရွယ်ချက်၊ တန်ဖိုး၊ သို့မဟုတ် လုပ်ဆောင်နိုင်စွမ်းသည် Software မှ သိသိသာသာ ဆင်းသက်လာသော မည်သည့်အလုပ် (vi) Software မှ ထုတ်လုပ်သော output၊ datasets၊ embeddings၊ model weights၊ သို့မဟုတ် feature stores များကို စားသုံး၊ ဝင်ရောက်၊ သို့မဟုတ် လုပ်ဆောင်သော မည်သည့်စနစ်။ Software ၏ မည်သည့်အစိတ်အပိုင်းကိုမဆို အရင်းအမြစ်ကုဒ်စာကြောင်း ၅၀ ထက်ပိုသော၊ တစ်ခုတည်းသော source file၊ အမည်ပေးထားသော function သို့မဟုတ် class တစ်ခု၊ data schema တစ်ခု၊ သို့မဟုတ် Software တွင် ဖော်ပြထားသော analytical algorithm သို့မဟုတ် workflow တစ်ခုခုကို အသုံးပြုခြင်းသည် Derivative Work တစ်ခုဖြစ်သည်။ ဤအတိုင်းအတာများသည် သာဓကအားဖြင့် အနိမ့်ဆုံးမျှသာဖြစ်ပြီး ဘေးကင်းသော ဆိပ်ကမ်းများ (safe harbors) မဟုတ်ပါ။ အသုံးပြုသူနှင့် Software ကြား ကြားခံ အဆင့်များ၊ wrapper များ၊ gateway များ၊ proxy များ၊ သို့မဟုတ် microservices များ၏ အရေအတွက်သည် သက်ဆိုင်မှုမရှိပါ။ စာသားမဟုတ်သော အစိတ်အပိုင်းများတွင် သိသိသာသာ ဆင်တူမှုသည် Derivative Work တစ်ခုဟု ယူဆရပြီး၊ စွပ်စွဲခံရသူအနေဖြင့် ရှင်းလင်းပြတ်သားသော သက်သေအထောက်အထားဖြင့် ပယ်ချရန် တာဝန်ရှိသည်။

- **"စီးပွားဖြစ်အသုံးပြုမှု (Commercial Use)"** ဆိုသည်မှာ Software (သို့) Modified Version (သို့) Derivative Work ကို တစ်စိတ်တစ်ပိုင်းဖြစ်စေ၊ တစ်ခုလုံးဖြစ်စေ၊ အမြတ်အစွန်းအတွက် အဖွဲ့အစည်း၊ အစိုးရအဖွဲ့အစည်း၊ အချုပ်အခြာအာဏာပိုင် အဖွဲ့အစည်း၊ ဗဟိုဘဏ်၊ ပြည်သူပိုင်လုပ်ငန်း၊ နိုင်ငံစုံအဖွဲ့အစည်း၊ အစိုးရနှင့် ဆက်စပ်သည့် အေဂျင်စီ၊ အကျိုးအမြတ်မယူသော အဖွဲ့အစည်း (non-profit)၊ ပရဟိတအဖွဲ့အစည်း၊ ဘာသာရေးအဖွဲ့အစည်း၊ NGO (ဝင်ငွေရှိမှု အခြေအနေ၊ အခွန်ကင်းလွတ်ခွင့် အခြေအနေ၊ သို့မဟုတ် အချုပ်အခြာအာဏာပိုင် ကိုယ်ခံခွင့် — သက်ဆိုင်ရာဥပဒေက ခွင့်ပြုသော အတိုင်းအတာအထိ ရှင်းလင်းစွာ စွန့်လွှတ်သည် — မခွဲခြားဘဲ) မှ သို့မဟုတ် ယင်းအဖွဲ့အစည်းအတွက် အလုပ်အကိုင်၊ စာချုပ်ချုပ်၊ အတိုင်ပင်ခံ၊ သို့မဟုတ် ဝန်ဆောင်မှုပေးခြင်းတွင် ပါဝင်သော မည်သည့်ပုဂ္ဂိုလ်မှ သုံးစွဲခြင်းကို ခေါ်သည်။ စီးပွားဖြစ်အသုံးပြုမှုတွင် အောက်ပါတို့ အကန့်အသတ်မရှိ ပါဝင်သည်:

  (က) အဖွဲ့အစည်းတစ်ခုအတွင်း **အတွင်းပိုင်းအသုံးပြုမှု**၊ Software ကို ပြင်ပအဖွဲ့အစည်းများသို့ ထုတ်ဖော်ပြသသည်ဖြစ်စေ၊ အဖွဲ့အစည်း၏ ကွန်ရက်၊ intranet၊ air-gapped ပတ်ဝန်းကျင်၊ သို့မဟုတ် အတွင်းပိုင်း Git repository တွင် လုံးဝကျန်ရှိသည်ဖြစ်စေ၊

  (ခ) ဝန်ထမ်းများ၊ ကန်ထရိုက်တာများ၊ သို့မဟုတ် အတိုင်ပင်ခံများ၏ အကဲဖြတ်ခြင်း၊ အယူအဆသက်သေပြခြင်း (proof-of-concept)၊ ရှေ့ပြေး (pilot)၊ သုတေသနနှင့် ဖွံ့ဖြိုးတိုးတက်ရေး၊ သရုပ်ပြခြင်း၊ အသွင်ကူးပြောင်းခြင်း (onboarding)၊ သို့မဟုတ် လေ့ကျင့်သင်ကြားခြင်းအတွင်း အသုံးပြုမှု၊

  (ဂ) လိုင်စင်ရသူ သို့မဟုတ် ၎င်း၏ တွဲဖက်ကုမ္ပဏီများ၏ မည်သည့်ထုတ်ကုန်၊ ဝန်ဆောင်မှု၊ သုတေသနထွက်ရှိမှု၊ စည်းကမ်းဆိုင်ရာ တင်သွင်းမှု၊ သို့မဟုတ် လုပ်ငန်းလည်ပတ်ရေး လုပ်ဆောင်ချက်ကို ပံ့ပိုးရန်၊ ဖြည့်စွက်ရန်၊ အစားထိုးရန်၊ သို့မဟုတ် ဖြည့်တင်းရန် အသုံးပြုမှု၊ ထိုထုတ်ကုန်၊ ဝန်ဆောင်မှု၊ သို့မဟုတ် ထွက်ရှိမှုကို ရောင်းချသည်ဖြစ်စေ၊ ဝင်ငွေထုတ်ပေးသည်ဖြစ်စေ မခွဲခြားဘဲ၊

  (ဃ) အမြတ်အစွန်းအတွက် အဖွဲ့အစည်း၊ အစိုးရအဖွဲ့အစည်း၊ သို့မဟုတ် အဖွဲ့အစည်းတစ်ခုခုမှ ပိုင်ဆိုင်၊ ငှားရမ်း၊ ထိန်းချုပ်၊ ငွေပေးချေ၊ သို့မဟုတ် ပံ့ပိုးထားသော ဟာ့ဒ်ဝဲ၊ ကွန်ရက်များ၊ virtual machines၊ containers၊ cloud အကောင့်များ၊ သို့မဟုတ် အခြေခံအဆောက်အအုံပေါ်တွင် မည်သည့်တပ်ဆင်ခြင်း သို့မဟုတ် လုပ်ဆောင်ခြင်းမဆို၊ အဖွဲ့အစည်းက တပ်ဆင်မှုကို ညွှန်ကြား၊ သိရှိ၊ သို့မဟုတ် အတည်ပြုသည်ဖြစ်စေ မခွဲခြားဘဲ၊ "ကိုယ်ရေးသုံး" ကင်းလွတ်ခွင့်သည် သဘာဝပုဂ္ဂိုလ်များအတွက်သာ သက်ရောက်ပြီး အလုပ်အကိုင်၏ လုပ်ငန်းစဉ်အပြင်ဘက်တွင် စီးပွားဖြစ်မဟုတ်သော ရည်ရွယ်ချက်များအတွက် ကိုယ်ပိုင်ပိုင် နှင့် ကိုယ်ပိုင်ငွေဖြင့်ဝယ်ယူထားသော ဟာ့ဒ်ဝဲကို အသုံးပြုသည့် သဘာဝပုဂ္ဂိုလ်များကိုသာ သက်ရောက်သည်၊

  (င) Modified Version မည်သည့်ကိုမဆို အသုံးပြုမှု၊ Fincept မှ ပံ့ပိုးပေးသော ဒေတာရင်းမြစ်များ၊ APIs၊ ပေါင်းစည်းမှုများ၊ သို့မဟုတ် ဝန်ဆောင်မှု endpoints များကို ဖယ်ရှား၊ အစားထိုး၊ ပြန်လည်ချိတ်ဆက်၊ သို့မဟုတ် လိုင်စင်ရသူ သို့မဟုတ် မည်သည့်ပြင်ပအဖွဲ့အစည်း၏ ဒေတာရင်းမြစ်များ၊ APIs, ပေါင်းစည်းမှုများ၊ သို့မဟုတ် ဝန်ဆောင်မှု endpoints များဖြင့် အစားထိုးထားသော ဗားရှင်းအပါအဝင်၊

  (စ) Software သို့ တိုက်ရိုက်ဖြစ်စေ၊ သွယ်ဝိုက်၍ဖြစ်စေ API ခေါ်ဆိုမှု၊ REST endpoint, RPC, gRPC, message queue, file drop, screen scrape, RPA bot, headless ခေါ်ဆိုမှု၊ သို့မဟုတ် အခြားမည်သည့် ကြားခံမှတစ်ဆင့် ဝင်ရောက်ခြင်း၊ Software ပါဝင်သော မည်သည့် container image သို့မဟုတ် virtual-machine image ကိုမဆို pull လုပ်ခြင်း၊ စတင်ခြင်း၊ လုပ်ဆောင်ခြင်း၊ mount လုပ်ခြင်း၊ သို့မဟုတ် အိမ်ရှင်အဖြစ် ဆောင်ရွက်ခြင်း အပါအဝင်၊

  (ဆ) အမြတ်အစွန်းအတွက် အဖွဲ့အစည်းတစ်ခုအတွင်း အတွင်းပိုင်း ပြန်လည်ဖြန့်ဝေခြင်း၊ အတွင်းပိုင်း Git repositories, artifact registries, package managers, mirrors, USB လွှဲပြောင်းမှု၊ သို့မဟုတ် peer-to-peer လွှဲပြောင်းမှုများ အပါအဝင်၊

  (ဇ) Software ကို လိုင်စင်ရသူ၏ ထုတ်ကုန်များနှင့် bundling, aggregation, integration, သို့မဟုတ် co-installation လုပ်ခြင်း၊ မည်သည့် open-source လိုင်စင်တွင် အသိအမှတ်ပြုထားသော "mere aggregation" ကင်းလွတ်ခွင့်ကိုမဆို မခွဲခြားဘဲ၊

  (ဇ) အမှုဆောင်ပညာရေး (executive education), MBA-ကော်ပိုရိတ် အစီအစဉ်များ၊ အခပေး လက်မှတ်အစီအစဉ်များ၊ ပံ့ပိုးထားသော သုတေသန ထွက်ရှိမှုများ၊ ကျွမ်းကျင်သူသက်သေအဖြစ် ဆောင်ရွက်မှု၊ နှင့် အခကြေးငွေ သို့မဟုတ် လျော်ကြေးငွေများ ထုတ်ပေးသော မည်သည့်သင်တန်း၊ လေ့ကျင့်ရေး၊ သို့မဟုတ် ဝန်ဆောင်မှုမဆို၊ ပညာရပ်ဆိုင်ရာ အဖွဲ့အစည်းမှ ပြုလုပ်သည့်တိုင်၊

  (ဇ) မည်သည့်ပုံစံဖြင့်မဆို ငွေကြေးအဖြစ် ပြောင်းလဲသော output (အခပေး သတင်းလွှာ၊ အတိုင်ပင်ခံ၊ အကြံပေး၊ ကျွမ်းကျင်သူသက်သေ၊ ဖွဲ့စည်းတည်ဆောက်ထားသော ထုတ်ကုန် (structured product) ဖန်တီးခြင်း၊ အချက်ပြများ (signal generation) ထုတ်ပေးခြင်း)၊ လက်ခံသူသည် အမြတ်အစွန်းအတွက် (for-profit) သို့မဟုတ် အကျိုးအမြတ်မယူသော (non-profit) အဖွဲ့အစည်းဖြစ်သည်ကို မခွဲခြားဘဲ။

  **ဒေတာရင်းမြစ်များ သို့မဟုတ် APIs ကို အစားထိုးခြင်းသည် မည်သည့်အခြေအနေတွင်မျှ စီးပွားဖြစ်အသုံးပြုမှုကို စီးပွားဖြစ်မဟုတ်သော အသုံးပြုမှုအဖြစ် ပြောင်းလဲခြင်း မရှိပါ။** **ကိုယ်ရေးသုံး ကင်းလွတ်ခွင့်ကို ကျဉ်းမြောင်းစွာ အဓိပ္ပာယ်ဖွင့်ဆိုပြီး သဘာဝပုဂ္ဂိုလ်တစ်ဦးမှ မည်သည့်စီးပွားရေး၊ အလုပ်အကိုင်၊ သို့မဟုတ် ဝင်ငွေဖန်တီးမှုလုပ်ငန်းစဉ်နှင့် လုံးဝမဆိုင်သော ရည်ရွယ်ချက်များအတွက် ကိုယ်ပိုင်ပိုင် ဟာ့ဒ်ဝဲကို အသုံးပြုခြင်းအတွက်သာ သက်ရောက်သည်။**

- **"Fincept အမှတ်တံဆိပ်များ (Fincept Marks)"** ဆိုသည်မှာ "Fincept", "Fincept Terminal", "Fincept Corporation" နှင့် အဆိုပါအမည်များ၏ အရောင်အဆင်း အတုခိုးမှု (colorable imitation)၊ အက္ခရာဖလှယ်မှု (transliteration)၊ အတိုကောက်၊ ဘာသာပြန်၊ သို့မဟုတ် အနကျအခံ သို့မဟုတ် ဆင်းသက်လာသော ပုံစံများကို ခေါ်သည်။

- **"Fincept ကုန်သွယ်အသွင် (Fincept Trade Dress)"** ဆိုသည်မှာ Fincept Terminal ၏ ထူးခြားသော အသွင်အပြင်နှင့် အဖုံအဖျား (look and feel) — screen အပြင်အဆင်များ၊ အရောင်ဖော်စပ် (color palette)၊ terminal command syntax, ticker conventions, function-code conventions, ကီးဘုတ်ဖြတ်လမ်းညွှန်များ (keyboard shortcut conventions)၊ dashboard widget ဝေါဟာရ (vocabulary)၊ နှင့် အလုံးစုံသော အမြင်ဆိုင်ရာ အမှတ်လက္ခဏာ (visual identity) တို့ကို ခေါ်သည်။

- **"လိုင်စင်ရှင် (Licensor)"** ဆိုသည်မှာ Fincept Corporation ဖြစ်သည်။ **"လိုင်စင်ရသူ (Licensee)"** ဆိုသည်မှာ ဤလိုင်စင်အောက်တွင် အခွင့်အရေးများ ကျင့်သုံးသော သဘာဝ သို့မဟုတ် ဥပဒေပုဂ္ဂိုလ်ကို ခေါ်သည်။

## ၄။ စီးပွားဖြစ်လိုင်စင် ဘယ်အချိန်မှာ လိုအပ်သနည်း

အောက်ပါတို့အားလုံးအတွက် စီးပွားဖြစ်လိုင်စင် လိုအပ်သည် — ဝင်ငွေ၊ အရွယ်အစား၊ သို့မဟုတ် ကြာချိန်အတွက် ကင်းလွတ်ခွင့်များ သို့မဟုတ် ခြွင်းချက်များ မရှိပါ။

- မည်သည့် စီးပွားရေးလုပ်ငန်းသုံးမှု၊ အခပေးဖြစ်စေ၊ အခမဲ့ဖြစ်စေ၊ ဝင်ငွေရှိသည်ဖြစ်စေ၊ မရှိသည်ဖြစ်စေ
- **မည်သည့်ကုမ္ပဏီ၊ ရန်ပုံငွေ၊ အစိုးရအဖွဲ့အစည်း၊ သို့မဟုတ် အမြတ်အစွန်းအတွက်အဖွဲ့အစည်းအတွင်း အတွင်းပိုင်းအသုံးပြုမှု** — Software ကို ပြင်ပအဖွဲ့အစည်းများသို့ ဘယ်တော့မှ ထုတ်ဖော်မပြသည့်တိုင်၊ Fincept APIs များကို ဖယ်ရှား သို့မဟုတ် အစားထိုးပြီးနောက်တိုင်၊
- မည်သည့်အဆင့်တွင်မဆို စတင်သည့်ကုမ္ပဏီများ (startups) — ဝင်ငွေမရသေးသော၊ ထုတ်ကုန်မရှိသေးသော၊ အစေ့မရွေးရသေးသော (pre-seed)၊ အကဲဖြတ်မှု၊ ရှေ့ပြေး၊ သို့မဟုတ် အယူအဆသက်သေပြခြင်း
- Hedge funds, proprietary trading firms, ဘဏ်များ၊ broker များ, asset managers, family offices, fintechs, နှင့် ငွေလဲလှယ်ရေးဌာနများ
- Software သို့မဟုတ် Derivative Work တစ်ခု၏ SaaS, PaaS, IaaS, သို့မဟုတ် မည်သည့် hosted, managed, သို့မဟုတ် cloud ကမ်းလှမ်းမှုမဆို
- Software သို့မဟုတ် Derivative Work တစ်ခု၏ white-label, OEM, အမည်ပြောင်းထားသော (rebranded), သို့မဟုတ် ပြန်လည်ရောင်းချသော ဖြန့်ဝေမှုများ
- စီးပွားဖြစ်အသုံးပြုမှုအတွက် Fincept APIs၊ ဒေတာရင်းမြစ်များ၊ သို့မဟုတ် ဝန်ဆောင်မှု endpoints များကို ဖယ်ရှား၊ အစားထိုး၊ ပိတ်၊ သို့မဟုတ် ပြန်လည်ချိတ်ဆက်ထားသော forks များ
- Software ကို ထည့်သွင်း သို့မဟုတ် မှီခိုသော အတိုင်ပင်ခံ၊ ပရော်ဖက်ရှင်နယ်ဝန်ဆောင်မှုများ၊ စနစ်ပေါင်းစည်းသူ (system integrator)၊ သို့မဟုတ် ကန်ထရိုက်တာများ၏ ပေးအပ်ချက်များ (deliverables)
- မည်သည့်အမြတ်အစွန်းအတွက်အဖွဲ့အစည်း၏ ဝန်ထမ်းများ သို့မဟုတ် ကန်ထရိုက်တာများအတွက် လေ့ကျင့်ရေး၊ အကဲဖြတ်ခြင်း၊ သရုပ်ပြခြင်း၊ သို့မဟုတ် အသွင်ကူးပြောင်းခြင်း (onboarding)
- Fincept Terminal သို့မဟုတ် လိုင်စင်ရှင်မှ ကမ်းလှမ်းသော မည်သည့်ထုတ်ကုန် သို့မဟုတ် ဝန်ဆောင်မှုနှင့် ယှဉ်ပြိုင်၊ အစားထိုး၊ သို့မဟုတ် သိသိသာသာ ပုံတူပွားသော မည်သည့်အသုံးပြုမှုမဆို

သင်မသေချာပါက၊ **စီးပွားဖြစ်လိုင်စင် လိုအပ်သည်ဟု ယူဆပြီး** support@fincept.in သို့ ဆက်သွယ်ပါ။

## ၅။ Repository ကို Cloning, Forking, သို့မဟုတ် ပြုပြင်ခြင်းက ပေးအပ်ခြင်း *မရှိသည်များ*

အများပြည်သူ repository တွင် တွဲထားသော AGPL-3.0 ထောက်ပံ့မှုသည် အသုံးပြုသူအား စီးပွားဖြစ်မဟုတ်သော ရည်ရွယ်ချက်များအတွက်သာ Software ကို AGPL-3.0 သတ်မှတ်ချက်များအောက်တွင် ကြည့်ရှု၊ လေ့လာ၊ ပြုပြင်၊ နှင့် ပြန်လည်ဖြန့်ဝေရန် ခွင့်ပြုသည်။ **၎င်းသည် အောက်ပါတို့အနက်မှ မည်သည့်အရာကိုမျှ ပေးအပ်ခြင်းမဟုတ်ပါ** — ယင်းအားလုံးသည် သီးခြား စီးပွားဖြစ်လိုင်စင် လိုအပ်သည်-

1. ကုမ္ပဏီတစ်ခု၊ ရန်ပုံငွေ၊ အစိုးရအဖွဲ့အစည်း၊ သို့မဟုတ် အမြတ်အစွန်းအတွက် အဖွဲ့အစည်းတစ်ခုအတွင်း Software ကို ပြင်ပ သို့မဟုတ် အတွင်းရေးရာ မည်သည့်ရည်ရွယ်ချက်အတွက်မဆို လုပ်ဆောင်ပိုင်ခွင့်
2. Software ကို မည်သည့်ပုံစံဖြင့်မဆို၊ ဝင်ငွေဖန်တီးမှု၊ စီးပွားရေး၊ ပရော်ဖက်ရှင်နယ်၊ သို့မဟုတ် လုပ်ငန်းလည်ပတ်ရေး လုပ်ဆောင်ချက်ကို ပံ့ပိုးရန် အသုံးပြုပိုင်ခွင့်
3. Fincept ၏ APIs, ဒေတာပေါင်းစည်းမှုများ၊ သို့မဟုတ် ဝန်ဆောင်မှု endpoints များကို ဖယ်ရှား၊ အစားထိုး၊ ပိတ်၊ သို့မဟုတ် ပြန်လည်ချိတ်ဆက်ပြီး ရရှိလာသော Modified Version သို့မဟုတ် Derivative Work ကို စီးပွားဖြစ် သို့မဟုတ် အတွင်းပိုင်း အသုံးပြုပိုင်ခွင့်
4. မည်သည့် fork, derivative, အမည်ပြောင်းထားသော၊ သို့မဟုတ် ဆက်ခံသော ထုတ်ကုန်တွင်မဆို Fincept အမှတ်တံဆိပ်များ သို့မဟုတ် Fincept Trade Dress ကို အသုံးပြု၊ ပြန်လည်ထုတ်လုပ်၊ သို့မဟုတ် အတုယူပိုင်ခွင့်
5. Software သို့မဟုတ် မည်သည့် Derivative Work ကိုမဆို ပြန်လည်လိုင်စင်ချပေး၊ လိုင်စင်ခွဲပေး၊ ရောင်းချ၊ ငှားရမ်း၊ သို့မဟုတ် အခပေးဝင်ရောက်ခွင့် ကမ်းလှမ်းပိုင်ခွင့်
6. Fincept မှ လည်ပတ်သော ဒေတာဝန်ဆောင်မှုများ၊ API endpoints, hosted အခြေခံအဆောက်အအုံ၊ သို့မဟုတ် တစ်မူထူးသော datasets များတွင် သော်လည်းကောင်း၊ ၎င်းတို့အပေါ် သော်လည်းကောင်း မည်သည့်အခွင့်အရေးမျှမရှိပါ။

လိုင်စင်သည် **codebase နှင့် ၎င်း၏ မည်သည့် Derivative Work ကိုမဆို တွဲနေသည်** — တိကျသော API ပေါင်းစည်းမှုများအတွက် မဟုတ်ပါ။ **Fincept APIs များကို သင်၏ကိုယ်ပိုင် — သို့မဟုတ် မည်သည့်ပြင်ပအဖွဲ့အစည်း၏ — ဖြင့် အစားထိုးခြင်းသည် လိုင်စင်လိုအပ်ချက်ကို ဖြတ်တောက် သို့မဟုတ် ငြိမ်းသွားစေခြင်း မရှိပါ။**

## ၆။ ကုန်အမှတ်တံဆိပ်နှင့် ကုန်သွယ်အသွင် (Trade Dress)

လိုင်စင်ရသူသည် လိုင်စင်ရှင်မှ ကြိုတင်ပြီး လိပ်မူဖြင့် ရှင်းလင်းစွာ ခွင့်ပြုချက်မရှိဘဲ မည်သည့်ထုတ်ကုန်၊ ဝန်ဆောင်မှု၊ repository၊ ဖြန့်ဝေမှု၊ သို့မဟုတ် စျေးကွက်ရှာဖွေရေးတွင်မဆို Fincept အမှတ်တံဆိပ်များကို အသုံးမပြုရ။ **မည်သည့် fork သို့မဟုတ် Derivative Work တွင်မဆို Fincept အမှတ်တံဆိပ်များကို ဖယ်ရှား၊ အစားထိုး၊ သို့မဟုတ် အမည်ပြောင်းခြင်းသည် နောက်ခံ Software တွင် လိုင်စင်ရှင်၏ အခွင့်အရေးများကို မပယ်ဖျက်နိုင်၊ ၎င်းသည် codebase နှင့် တွဲနေသော လိုင်စင်လိုအပ်ချက်များကို မဖြတ်တောက်နိုင်ပါ။**

လိုင်စင်ရသူသည် Fincept အမှတ်တံဆိပ်များ၏ တန်ဖိုးသည် စာသားအတိုင်း ကုန်အမှတ်တံဆိပ် စာသားတွင်သာမက Fincept Trade Dress တွင် တည်ရှိကြောင်း အသိအမှတ်ပြုသည်။ လိုင်စင်ရသူသည် မည်သည့် fork, derivative, သို့မဟုတ် ဆက်ခံသော ထုတ်ကုန်တွင်မဆို၊ စာသားအတိုင်း Fincept အမှတ်တံဆိပ်များကို ဖယ်ရှား သို့မဟုတ် အစားထိုးထားသည့်တိုင်၊ ရင်းမြစ် (source) သို့မဟုတ် ဆက်နွယ်မှု (affiliation) နှင့်ပတ်သက်၍ ရှုပ်ထွေးမှု၊ အမှား၊ လှည့်စားမှု၊ ဆက်စပ်မှု (association)၊ သို့မဟုတ် ကနဦးအကျိုးစီးပွားဆိုင်ရာ ရှုပ်ထွေးမှု (initial-interest confusion) ဖြစ်နိုင်ခြေရှိသော ပုံစံဖြင့် Fincept Trade Dress ကို ပုံတူမပွားရ။ **UI စာသား၊ terminal commands, function codes, သို့မဟုတ် screen အမည်များကို အခြားဘာသာစကား သို့မဟုတ် အက္ခရာသို့ ဘာသာပြန်ခြင်းသည် trade-dress အကာအကွယ်ကို မပယ်ဖျက်ပါ။** တစ်ဦးချင်း screen အမည်များ၊ function codes, terminal commands, သို့မဟုတ် ကီးဘုတ်ဖြတ်လမ်းများကို အမည်ပြောင်းခြင်း၊ အတိုကောက်လုပ်ခြင်း၊ သို့မဟုတ် အစားထိုးခြင်းသည် နောက်ခံ *စနစ်* ၏ ညီညွတ်မှုများကို သိသိသာသာ ထိန်းသိမ်းထားပါက trade-dress တာဝန်ယူမှုမှ ကင်းလွတ်ခွင့်မရှိပါ။ သိသိသာသာ ဆင်တူသော Fincept Trade Dress ကို ထိန်းသိမ်းထားသော မည်သည့်ထုတ်ကုန်မဆို ဤလိုင်စင်၏ ရည်ရွယ်ချက်များအားလုံးအတွက် Derivative Work အဖြစ် သတ်မှတ်ခံရမည်ဖြစ်ပြီး ၎င်း၏ စီးပွားဖြစ်အသုံးပြုမှုအားလုံးသည် လိုင်စင်ရှင်ထံမှ စီးပွားဖြစ်လိုင်စင် လိုအပ်သည်။

**နှိုင်းယှဉ်ကြော်ငြာ (Comparative Advertising)**။ အမြတ်အစွန်းအတွက် ပြိုင်ဘက် သို့မဟုတ် ၎င်း၏ ကိုယ်စားလှယ်များက စျေးကွက်ရှာဖွေရေး၊ အရောင်းပစ္စည်းများ၊ သရုပ်ပြမှုများ၊ RFP တုံ့ပြန်မှုများ၊ ဖန်သားပြင်ဓာတ်ပုံများ၊ သို့မဟုတ် ထုတ်ကုန်နှိုင်းယှဉ်မှုများတွင် Fincept အမှတ်တံဆိပ်များ သို့မဟုတ် Fincept Trade Dress ကို မည်သည့်အသုံးပြုမှုမဆို အမှတ်တံဆိပ်များ၏ စီးပွားဖြစ်အသုံးပြုမှုဖြစ်ပြီး၊ မည်သည့် nominative-fair-use သို့မဟုတ် နှိုင်းယှဉ်ကြော်ငြာ ကာကွယ်မှုကိုမဆို မခွဲခြားဘဲ၊ ရှင်းလင်းသော လိပ်မူဖြင့် ခွင့်ပြုချက်မရှိလျှင် တားမြစ်သည်။

**လွတ်လပ်စွာ ဖန်တီးခြင်း၏ တာဝန် (Burden of Independent Creation)**။ Derivative Work တောင်းဆိုချက်ကို ဆန့်ကျင်သော ကာကွယ်ရေးအနေဖြင့် လိုင်စင်ရသူ သို့မဟုတ် မည်သည့် Third-Party Developer က လွတ်လပ်စွာ ဖန်တီးခြင်းကို အခိုင်အမာပြောဆိုပါက၊ လွတ်လပ်စွာ ဖန်တီးခြင်းကို သက်သေပြရန် တာဝန်သည် စွပ်စွဲခံရသူ (alleged infringer) တွင် တည်ရှိပြီး၊ ခေတ်ပြိုင် ဒီဇိုင်းစာရွက်စာတမ်းများ၊ ပြီးပြည့်စုံသော commit သမိုင်း၊ ဝန်ထမ်းအသိအမှတ်ပြုလက်မှတ်များ၊ နှင့် clean-room ဖွံ့ဖြိုးတိုးတက်မှုမှတ်တမ်းများဖြင့် သက်သေပြရမည်။ ယင်းအားလုံးကို လိုအပ်ပါက လိုင်စင်ရှင်ထံ ထုတ်ဖော်ပြသရမည်။

ရင်းမြစ်ရနိုင်သောပုံစံ (source-available form) ဖြင့် အများသူငှာ ဖြန့်ဝေသော မည်သည့် Derivative Work မဆို အောက်ပါ အသိပေးချက်ကို ထင်ရှားစွာ ပြသရမည်: *"Derived from Fincept Terminal © Fincept Corporation; use of this Derivative Work for Commercial Use requires a separate Commercial License from Fincept Corporation, support@fincept.in."*

## ၈။ မူပိုင်ခွင့် (Patent) ထောက်ပံ့မှုနှင့် မူပိုင်ခွင့် လက်စားချေမှု (Patent Retaliation)

ဤလိုင်စင်၏ သတ်မှတ်ချက်များနှင့်အညီ၊ လိုင်စင်ရှင်သည် လိုင်စင်ရသူအား ဤနေရာတွင် ခွင့်ပြုထားသည့်အတိုင်း Software ကို ပြုလုပ်၊ လုပ်ကိုင်၊ အသုံးပြု၊ ကမ်းလှမ်း၊ ရောင်းချ၊ တင်သွင်း၊ နှင့် တင်သွင်းပြီးဖြစ်စေရန်အတွက် လိုင်စင်ရှင်၏ မူပိုင်ခွင့်တောင်းဆိုချက်များအောက်တွင် သီးသန့်မဟုတ်သော (non-exclusive)၊ ကမ္ဘာတစ်ဝှမ်း၊ အခကြေးငွေပါသော (royalty-bearing) လိုင်စင်ကို ပေးအပ်သည်။

**လိုင်စင်ရသူ၊ သို့မဟုတ် လိုင်စင်ရသူကို ထိန်းချုပ်သော၊ ထိန်းချုပ်ခံရသော၊ သို့မဟုတ် လိုင်စင်ရသူနှင့် ဘုံထိန်းချုပ်မှုအောက်ရှိ မည်သည့်အဖွဲ့အစည်းကမဆို Software၊ ၎င်း၏အသုံးပြုမှု၊ သို့မဟုတ် ၎င်းအတွက် ပြုလုပ်သော မည်သည့် ထည့်ဝင်မှု (contribution) သည် တိုက်ရိုက် (direct) သို့မဟုတ် ဖြည့်စွက် (contributory) မူပိုင်ခွင့်ချိုးဖောက်မှုဖြစ်သည်ဟု စွပ်စွဲကာ မည်သည့်အဖွဲ့အစည်းကိုမဆို မူပိုင်ခွင့်တရားစွဲဆိုမှု (cross-claim သို့မဟုတ် counterclaim အပါအဝင်) စတင်ပါက၊ ထိုတရားစွဲဆိုမှုကို တင်သည့်ရက်စွဲတွင် ဤလိုင်စင်အောက်ရှိ လိုင်စင်ရသူအား ပေးအပ်ထားသော မူပိုင်ခွင့်လိုင်စင်များ ရပ်စဲသည်။** လိုင်စင်ရသူမှ မိတ်ဆက်သော ပြုပြင်မွမ်းမံမှုများမှ ပေါ်ပေါက်လာသော မူပိုင်ခွင့်တောင်းဆိုချက်များကို လိုင်စင်မပေးပါ။

## ၉။ စစ်ဆေးခွင့် (Audit Rights)

လိုင်စင်ရှင်သည် စာဖြင့် အကြောင်းကြားချက် ရက်ပေါင်း ၃၀ (ကြိုတင်ရက်ထည့်) ပေးပြီး၊ ဆယ့်နှစ်လတစ်ကြိမ်ထက် မပိုသော (လိုင်စင်ရှင်သည် ချိုးဖောက်မှုကို ကျိုးကြောင်းဆီလျော်စွာ သံသယရှိသည့်အခါမှအပ) အချိန်တွင်၊ လျှို့ဝှက်မှုဖြင့် ချုပ်နှောင်ထားသော အပြန်အလှန် သဘောတူညီသည့် လွတ်လပ်သော စာရင်းစစ်အဖွဲ့ (auditor) မှတစ်ဆင့် လိုင်စင်ရသူ၏ Software အသုံးပြုမှုကို စစ်ဆေးနိုင်သည်။ လိုင်စင်ရသူသည် အသုံးပြုမှုမှတ်တမ်းများ၊ အသုံးပြုသူအရေအတွက်၊ အခြေခံအဆောက်အအုံ စာရင်း၊ နှင့် Software သို့မဟုတ် Derivative Works ပါရှိသော ရင်းမြစ် repositories များကို ဝင်ရောက်ခွင့် အပါအဝင် ကျိုးကြောင်းဆီလျော်သော ပူးပေါင်းဆောင်ရွက်မှုကို ပေးအပ်ရမည်။ စစ်ဆေးချက်က ငါးရာခိုင်နှုန်း (၅%) ထက်ပိုသော လျော့ပေးချေမှု သို့မဟုတ် ခွင့်ပြုချက်မရှိသော စီးပွားဖြစ်အသုံးပြုမှုကို ဖော်ထုတ်ပါက၊ လိုင်စင်ရသူသည် (က) ထိုအချိန်က သုံးစွဲဆဲနှုန်းထားဖြင့် နောက်ကြောင်းပြန် တွက်ချက်သော မပေးရသေးသော အခကြေးငွေများ၊ (ခ) တစ်နှစ်လျှင် ၁၈% သို့မဟုတ် ဥပဒေက ခွင့်ပြုသော အများဆုံးနှုန်းထား (အနိမ့်ဆုံးဖြစ်သော) အတိုင်း အတိုး၊ နှင့် (ဂ) စစ်ဆေးခြင်း၏ ကျိုးကြောင်းဆီလျော်သော ကုန်ကျစရိတ်များကို ပေးချေရမည်။

**နှစ်စဉ် ကိုယ်တိုင် အသိအမှတ်ပြုခြင်း (Annual Self-Attestation)**။ Software ၏ အမြတ်အစွန်းအတွက် အသုံးပြုသူတိုင်းသည် ပြက္ခဒိန်နှစ်တစ်နှစ်၏ ဇန်နဝါရီလ ၃၁ ရက်နေ့ နောက်ဆုံးထားပြီး၊ ၎င်း၏ လိုင်စင်အခြေအနေ၊ အသုံးပြုသူအရေအတွက်၊ အသုံးပြုမှုအရေအတွက် (deployments)၊ နှင့် Software ကိုသုံးသော ဥပဒေပုဂ္ဂိုလ်အမည်(များ) ကိုဖော်ပြသော စာဖြင့်ရေးသားသည့် အသိအမှတ်ပြုချက်ကို support@fincept.in သို့ တင်သွင်းရမည်။ အသိအမှတ်ပြုချက် တင်သွင်းရန် ပျက်ကွက်ခြင်းသည် ပုဒ်မ 13 တွင်ဖော်ပြထားသော ပျက်စီးဆုံးရှုံးမှုကြေး (liquidated damages) ကို ဖြစ်ပေါ်စေသော အလိုအလျောက် ချိုးဖောက်မှု (per se breach) ဖြစ်သည်။

**ကိုယ်ရေးကိုယ်တာနှင့် ကုန်သွယ်ရေးလျှို့ဝှက်ချက် (Trade-Secret) ကန့်သတ်ချက်များ**။ အကျုံးဝင်သော ဒေတာကာကွယ်ရေးဥပဒေ (EU အထွေထွေ ဒေတာကာကွယ်ရေး စည်းမျဉ်း (GDPR)၊ ဒီဂျစ်တယ် ကိုယ်ရေးအချက်အလက် ကာကွယ်ရေးအက်ဥပဒေ၊ 2023 (အိန္ဒိယ)၊ သို့မဟုတ် အလားတူ ဥပဒေများ အပါအဝင်) သည် ကိုယ်ရေးအချက်အလက် (personal data) ထုတ်ဖော်မှုကို ကန့်သတ်ပါက၊ လိုင်စင်ရှင်သည် လိုက်နာမှုကို စစ်ဆေးရန် လုံလောက်သော အနည်းဆုံး အချက်အလက် — (i) စုစုပေါင်း အသုံးပြုသူအရေအတွက်၊ (ii) စုစုပေါင်း အသုံးပြုမှုအရေအတွက်၊ (iii) အဖွဲ့အစည်း၏ ဥပဒေအမည်၊ နှင့် (iv) hash လုပ်ထားသော code-base လက်ဗွေများ — ပါဝင်သော အမည်မသိ (anonymized) သို့မဟုတ် ပရိယာယ် (pseudonymized) ဒေတာကို လက်ခံရမည်။ စစ်ဆေးမှုတုံ့ပြန်မှုအတွက် မည်သည့် ဒေတာကာကွယ်ရေး လိုက်နာမှုကုန်ကျစရိတ်ကိုမဆို လိုင်စင်ရသူက ကျခံရမည်။ လိုင်စင်ရသူက ၎င်း၏ရင်းမြစ်ကုဒ်ကို ကုန်သွယ်ရေးလျှို့ဝှက်ချက်အဖြစ် အကာအကွယ်ပေးသည်ဟု အခိုင်အမာဆိုပါက၊ လိုင်စင်ရှင်သည် ရလဒ်များကို လိုက်နာမှုအခြေအနေအဖြစ်သာ ပေးအပ်သည့် အကျိုးဆောင်-ဖောက်သည်အခွင့်အရေး (attorney-client privilege) အောက်တွင် လုပ်ကိုင်သော အပ်ငွေထိန်း (escrow agent) မှ ကုဒ်ပြန်လည်သုံးသပ်ခြင်းကို လက်ခံမည်။ ပြင်ပအဖွဲ့အစည်းများထံ လျှို့ဝှက်ရန်တာဝန် (NDAs with Licensee's clients) များသည် လိုင်စင်ရှင်၏ စစ်ဆေးခွင့်ကို မထိခိုက်စေရ။ လိုင်စင်ရသူသည် လိုအပ်သော ဖောက်သည်ခွင့်ပြုချက်ရယူရန် စီးပွားဖြစ် ကျိုးကြောင်းဆီလျော်သော အားထုတ်မှု (commercially reasonable efforts) ကို ပြုလုပ်ရမည် သို့မဟုတ် အနီရောင်ဖြတ်ထားသော (redacted) ညီမျှသော အချက်အလက်ကို ပေးအပ်ရမည်။ အကျုံးဝင်သော နိုင်ငံခြား ပိတ်ဆို့ခြင်း ဥပဒေ (foreign blocking statute) က ထုတ်ဖော်မှုကို စစ်မှန်စွာ တားဆီးပါက၊ လိုင်စင်ရသူသည် ခွင့်ပြုထားသော တရားစီရင်ပိုင်ခွင့်မှ ညီမျှသော အချက်အလက်ကို ပေးအပ်ရမည် သို့မဟုတ် ပုဒ်မ 13 အရ စစ်ဆေးခြင်းအစား ပျက်စီးဆုံးရှုံးမှုကြေး (liquidated damages) ပေးချေရမည်။

**တတိယပုဂ္ဂိုလ် ဆော့ဖ်ဝဲပညာရှင် (Third-Party Developers) ထုတ်ဖော်ခြင်း**။ လိုင်စင်ရှင်၏ စာဖြင့်တောင်းဆိုမှုအရ၊ လိုင်စင်ရသူသည် လုပ်ငန်းရက် ဆယ်ရက် (၁၀) အတွင်း အောက်ပါတို့ကို ပေးအပ်ရမည်- (က) Software နှင့် ပတ်သက်သော မည်သည့် Third-Party Developer ၏ ဥပဒေအမည်နှင့် ဆက်သွယ်ရန်အသေးစိတ်၊ (ခ) လုပ်ဆောင်မှု၏ နယ်ပယ်၊ (ဂ) စီးပွားရေးဆိုင်ရာ စျေးနှုန်းသတ်မှတ်ချက်များအထိသာ အနီရောင်ဖြတ်ထားသော (redacted) သက်ဆိုင်ရာ စာချုပ်ချုပ် သို့မဟုတ် လုပ်ငန်းနယ်ပယ် (statement of work) ၏ မိတ္တူများ၊ နှင့် (ဃ) Software သို့ ဝင်ရောက်ခွင့်ရှိသော repositories များ၊ အသုံးပြုမှုနေရာများ၊ နှင့် ဝန်ထမ်းများ၏စာရင်း။

## ၁၀။ ရပ်စဲခြင်းနှင့် ရှင်သန်ခြင်း (Termination and Survival)

ဤလိုင်စင်သည် လိုင်စင်ရသူမှ မည်သည့်ပြဌာန်းချက်ကိုမဆို ချိုးဖောက်သည်နှင့် အလိုအလျောက် အကြောင်းကြားစရာမလိုဘဲ ရပ်စဲသည်။ ရပ်စဲခြင်းသည် လိုင်စင်ရသူအား မပေးရသေးသော အခကြေးငွေများ အပါအဝင် ကြွေးကျန်ရှိသော တာဝန်ဝတ္တရားများမှ လွှတ်ငြိမ်းချမ်းသာခွင့် မပေးပါ။ ရပ်စဲပြီးနောက်၊ လိုင်စင်ရသူသည် အသုံးပြုမှုအားလုံးကို ချက်ချင်းရပ်တန့်ရမည်၊ ၎င်း၏ သိမ်းဆည်းမှု သို့မဟုတ် ထိန်းချုပ်မှုအောက်ရှိ Software နှင့် Derivative Works အားလုံး၏ မိတ္တူအားလုံးကို ဖျက်ဆီးရမည်၊ နှင့် ထိုသို့ဖျက်ဆီးကြောင်း စာဖြင့် ဆယ့်ငါးရက် (၁၅) အတွင်း အသိအမှတ်ပြုရမည်။

အောက်ပါ ပြဌာန်းချက်များသည် ရပ်စဲပြီးနောက် ရှင်သန်သည်- ပုဒ်မ ၃ (အဓိပ္ပာယ်ဖွင့်ဆိုချက်များ)၊ ၅ (မသွယ်ဝိုက်သော ထောက်ပံ့မှု)၊ ၆ (ကုန်အမှတ်တံဆိပ်နှင့် ကုန်သွယ်အသွင်)၊ ၈ (မူပိုင်ခွင့် လက်စားချေမှု)၊ ၉ (စစ်ဆေးခွင့်၊ ရပ်စဲပြီး နှစ်နှစ်အတွက်)၊ ၁၁ (အာမခံချက်မှ ငြင်းဆိုခြင်း)၊ ၁၂ (အတည်ဖြစ်စေခြင်း)၊ ၁၃ (ပျက်စီးဆုံးရှုံးမှုကြေးနှင့် ပြစ်ဒဏ်များ)၊ ၁၄ (ပူးတွဲနှင့် တစ်ဦးချင်းတာဝန်ယူမှု)၊ ၁၅ (တာဝန်ယူမှု ကန့်သတ်ချက်)၊ ၁၆ (အုပ်ချုပ်သည့်ဥပဒေနှင့် တရားစီရင်ပိုင်ခွင့်)၊ နှင့် ဤပုဒ်မ ၁၀။

## ၁၂။ အတည်ဖြစ်စေခြင်း (Enforcement)

လိုင်စင်ရှင်သည် Software, Modified Versions, နှင့် Derivative Works များ၏ လိုင်စင်မဲ့ စီးပွားဖြစ်အသုံးပြုမှုအတွက် အများပြည်သူ code repositories, container registries, application stores, cloud marketplaces, SaaS ပလက်ဖောင်းများ, နှင့် အမှတ်တံဆိပ်မှတ်ပုံတင်ခြင်းများကို တက်ကြွစွာ စောင့်ကြည့်သည်။ လိုင်စင်ရှင်သည် ကြိုတင်အကြောင်းကြားချက်မပါဘဲ အောက်ပါတို့ကို ဆောင်ရွက်ပိုင်ခွင့် ရှိသည်-

1. US Digital Millennium Copyright Act (17 U.S.C. § 512), Information Technology Act, 2000 (India) §§ 65 and 66, Copyright Act, 1957 (India) §§ 63–63B, နှင့် အခြားတရားစီရင်ပိုင်ခွင့်များရှိ ညီမျှသော ဥပဒေများအောက်တွင် ဖယ်ရှားရေးအကြောင်းကြားစာ (takedown notices) ထုတ်ပြန်ခြင်း;
2. ချိုးဖောက်နေသော အဖွဲ့အစည်းများနှင့် ၎င်းတို့၏ အရာရှိများ၊ ဒါရိုက်တာများ၊ နှင့် ရှေ့နေများထံ ရပ်တန့်ရန် နှင့် ရှောင်ကြဉ်ရန် စာ (cease-and-desist letters) ပေးပို့ခြင်း;
3. တားမြစ်မိန့် (injunctive) နှင့် ငွေကြေးဆိုင်ရာ ကုစားမှုများ၊ ဥပဒေအရ ပျက်စီးဆုံးရှုံးမှုကြေးများ (statutory damages)၊ အမြတ်အစွန်း လွှဲပြောင်းပေးအပ်ခြင်း (disgorgement of profits)၊ နှင့် ပထမဆုံး ခွင့်ပြုချက်မရှိသော အသုံးပြုမှုရက်စွဲမှ နောက်ကြောင်းပြန် တွက်ချက်သော ထိုအချိန်က သုံးစွဲဆဲ စီးပွားဖြစ်လိုင်စင်နှုန်းထားဖြင့် မပေးရသေးသော လိုင်စင်အခကြေးငွေများ ပြန်လည်ရယူခြင်း၊ အပြင် အတိုး၊ ရှေ့နေအခကြေးငွေ၊ နှင့် ကုန်ကျစရိတ်များ ရယူရန် တရားစွဲခြင်း;
4. Fincept အမှတ်တံဆိပ်များ သို့မဟုတ် Fincept Trade Dress ကို အလွဲသုံးစားမှုအတွက် ကုန်အမှတ်တံဆိပ်ချိုးဖောက်မှု၊ ဖြတ်သန်းခြင်း (passing-off)၊ နှင့် မတရားယှဉ်ပြိုင်မှု (unfair-competition) တောင်းဆိုချက်များ တင်သွင်းခြင်း;
5. Copyright Act, 1957 (India) §§ 63–63B နှင့် ညီမျှသော နိုင်ငံခြားဥပဒေများအောက်တွင် ရာဇဝတ်ဆိုင်ရာ တိုင်ကြားချက်များ တင်သွင်းခြင်း။

သင်သည် လိုင်စင်မရှိဘဲ Software ကို စီးပွားဖြစ် အသုံးပြုနေနိုင်သည်ဟု ယုံကြည်ပါက၊ ကောင်းမွန်သော ယုံကြည်ချက်ဖြင့် ဖြေရှင်းရန် လမ်းကြောင်းအတွက် **support@fincept.in** သို့ ဆက်သွယ်ပါ။

## ၁၃။ ခွင့်ပြုချက်မရှိသော အသုံးပြုမှုအတွက် ပျက်စီးဆုံးရှုံးမှုကြေးနှင့် ပြစ်ဒဏ်များ (Liquidated Damages and Penalties)

**ဆုံးရှုံးမှု၏ စစ်မှန်သော ကြိုတင်ခန့်မှန်းချက် — နည်းစနစ် (Methodology)** ။ ပါတီများသည် အောက်ပါတို့ကို ရှင်းလင်းစွာ အသိအမှတ်ပြုပြီး သဘောတူသည်-

(i) ခွင့်ပြုချက်မရှိသော စီးပွားဖြစ်အသုံးပြုမှုမှ ပေါ်ပေါက်လာသော အမှန်တကယ် ပျက်စီးဆုံးရှုံးမှုများကို အရေအတွက်အရ ခန့်မှန်းရန် ပင်ကိုယ်အားဖြင့် ခက်ခဲသည် — အကြောင်းမှာ ၎င်းတို့တွင် (က) နောက်မှ လိုင်စင်သွင်းခြင်းဖြင့် ပြန်လည်မရရှိနိုင်သော စီးပွားဖြစ်လိုင်စင်ဝင်ငွေ ဆုံးရှုံးခြင်း၊ (ခ) ပြိုင်ဆိုင်မှုအားသာချက်နှင့် စျေးကွက်ဝေစု ဆုံးရှုံးခြင်း၊ (ဂ) open-source codebase ကို ငွေကြေးထောက်ပံ့သော dual-licensing စီးပွားရေးပုံစံ ဆုံးရှုံးခြင်း၊ (ဃ) ကျယ်ပြန့်သော ခွင့်ပြုချက်မရှိသော အတွင်းပိုင်းအသုံးပြုမှုများကို ရှာဖွေတွေ့ရှိခြင်း၊ စောင့်ကြည့်ခြင်း၊ စုံစမ်းစစ်ဆေးခြင်း၊ နှင့် အတည်ဖြစ်စေခြင်း ကုန်ကျစရိတ်များ၊ နှင့် (င) ဂုဏ်သတင်းနှင့် နာမည်ကောင်း ဆုံးရှုံးခြင်း တို့ပါဝင်သည်၊

(ii) အောက်ပါတွင်ဖော်ပြထားသော ပမာဏများသည် အောက်ပါတို့ကို ရည်ညွှန်းချက်အဖြစ် တွက်ချက်သော **ဆုံးရှုံးမှု၏ စစ်မှန်သော ကြိုတင်ခန့်မှန်းချက်** ဖြစ်သည်- (က) ထုတ်ဝေထားသော နှစ်စဉ် စီးပွားဖြစ်လိုင်စင်ကြေး USD ၁၀,၂၀၀၊ (ခ) ချိုးဖောက်မှုတစ်ခုလျှင် ပျမ်းမျှ လုပ်ငန်းဆိုင်ရာ လိုက်နာမှုနှင့် ရှာဖွေတွေ့ရှိမှု ကုန်ကျစရိတ်၊ (ဂ) ငွေကြေးဆော့ဖ်ဝဲလုပ်ငန်း စံသတ်မှတ်ချက်များကို အခြေခံ၍ ခွင့်ပြုချက်မရှိသော အသုံးပြုမှုတစ်ခုလျှင် ပျမ်းမျှ ဝင်ငွေ လွှဲပြောင်းခြင်း (revenue diversion)၊ နှင့် (ဃ) အတွင်းပိုင်း ကော်ပိုရိတ် fork တစ်ခု နှစ်ပေါင်းများစွာ သတိမထားမိဘဲ လွတ်မြောက်သွားသည့် လူသိများသော "ဓာတ်ခွဲခန်း ယိုစိမ့်မှု (lab leak)" ပုံစံကို ကာကွယ်ရန် ကျိုးကြောင်းဆီလျော်စွာ လိုအပ်သော အတားအဆီးတန်ဖိုး၊

(iii) ပမာဏများသည် အိန္ဒိယစာချုပ်အက်ဥပဒေ (Indian Contract Act), 1872 ၏ ပုဒ်မ 74 အဓိပ္ပာယ်အတွင်း *အကြောက်လွန်ပြစ်ဒဏ် (penalty in terrorem)* မဟုတ်ဘဲ ကျိုးကြောင်းဆီလျော်သော လျော်ကြေးငွေ ဖြစ်သည်။

(iv) ပါတီများသည် ဤပမာဏများ၏ ကျိုးကြောင်းဆီလျော်မှုကို သဘောတူညီပြီး ၎င်းတို့သည် ပြစ်ဒဏ်တစ်ရပ်၊ လွန်ကဲသည်၊ သို့မဟုတ် *အကြောက်လွန်ပြစ်ဒဏ်* ဖြစ်သည်ဟူသော မည်သည့်ကာကွယ်ရေးကိုမဆို စွန့်လွှတ်သည်၊ သို့သော် ထိုသို့စွန့်လွှတ်ခြင်းသည် အတင်းအကြပ် ဥပဒေ (mandatory law) အရ အတည်မဖြစ်နိုင်ပါက ချွင်းချက်ဖြစ်သည်။

Software, Modified Version, သို့မဟုတ် Derivative Work တစ်ခုခုကို ခွင့်ပြုချက်မရှိဘဲ စီးပွားဖြစ်အသုံးပြုမှုတွင် ပါဝင်သော မည်သည့်ပါတီမဆို ဥပဒေ သို့မဟုတ် တရားမျှတမှု (equity) တွင် ရရှိနိုင်သော အခြား ကုစားနည်းအားလုံးအပြင် အောက်ပါ ပျက်စီးဆုံးရှုံးမှုကြေးအတွက် လိုင်စင်ရှင်ထံ တာဝန်ယူရမည်-

ပျက်စီးဆုံးရှုံးမှုကြေးများအပြင်၊ လိုင်စင်ရှင်သည် အောက်ပါတို့ကို ပြန်လည်ရယူပိုင်ခွင့် ရှိသည်:

2. ချိုးဖောက်သူမှ ခွင့်ပြုချက်မရှိသော အသုံးပြုမှုမှ ရရှိသော အမြတ်အစွန်း၊ ဝင်ငွေ၊ ကုန်ကျစရိတ်ချွေတာမှု၊ နှင့် အခြားသော စီးပွားရေးအကျိုးအမြတ်အားလုံးကို လွှဲပြောင်းပေးအပ်ခြင်း (disgorgement);
3. ရှေ့နေအခကြေးငွေ၊ တရားရုံးအခကြေးငွေ၊ ကျွမ်းကျင်သူအခကြေးငွေ၊ စုံစမ်းစစ်ဆေးရေးကုန်ကျစရိတ်၊ နှင့် အခြား အတည်ဖြစ်စေရေး ကုန်ကျစရိတ်အားလုံး;

5. အထူးသဖြင့် ရည်ရွယ်ချက်ရှိရှိ (willful) သို့မဟုတ် ထပ်ခါတလဲလဲ ချိုးဖောက်မှုများအတွက် ဥပဒေက ခွင့်ပြုပါက အပြစ်ပေး (punitive) နှင့် သာဓကပြ (exemplary) ပျက်စီးဆုံးရှုံးမှုများ။

အထက်တွင်ဖော်ပြထားသော ပျက်စီးဆုံးရှုံးမှုကြေးများသည် **ချိုးဖောက်မှုတစ်ခုစီအတွက်၊ စုပေါင်း (cumulative)** ဖြစ်ပြီး **ခွင့်ပြုချက်မရှိသော အသုံးပြုမှု၏ တစ်နှစ်စီအတွက်** အကျုံးဝင်သည်။ ပျက်စီးဆုံးရှုံးမှုကြေးများသည် လိုင်စင်ကြေးတစ်ရပ် မဟုတ်ပါ၊ ခွင့်ပြုချက်မရှိသော အသုံးပြုမှုကို နောက်ကြောင်းပြန်၍ တရားဝင်ဖြစ်စေခြင်း မရှိပါ၊ နှင့် တားမြစ်မိန့် (injunctive relief)၊ ရာဇဝတ်တရားစွဲဆိုမှု၊ သို့မဟုတ် အခြား ကုစားနည်းများကို ရှာဖွေရန် လိုင်စင်ရှင်၏ အခွင့်အရေးကို စွန့်လွှတ်ခြင်း မဟုတ်ပါ။

## ၁၄။ စာချုပ်ချုပ်သော ကုမ္ပဏီများ၊ ဆော့ဖ်ဝဲပညာရှင်များ၊ ပေါင်းစည်းသူများနှင့် တွဲဖက်ကုမ္ပဏီများ၏ ပူးတွဲနှင့် တစ်ဦးချင်းတာဝန်ယူမှု (Joint and Several Liability)

**တတိယပုဂ္ဂိုလ် — ဆော့ဖ်ဝဲဖွံ့ဖြိုးတိုးတက်ရေးကုမ္ပဏီများ၊ စနစ်ပေါင်းစည်းသူများ၊ IT အတိုင်ပင်ခံကုမ္ပဏီများ၊ ကန်ထရိုက်တာများ၊ အလွတ်တန်းသမားများ၊ ပြင်ပရင်းမြစ်ရှာဖွေသူများ (outsourcing vendors), offshore ဖွံ့ဖြိုးတိုးတက်ရေးစင်တာများ၊ white-label ဆော့ဖ်ဝဲပညာရှင်များ၊ သို့မဟုတ် တန်ဖိုးထပ်တိုး ပြန်လည်ရောင်းချသူများ (value-added resellers) (စုပေါင်း၍ "Third-Party Developers" ဟုခေါ်) — ကို Software, Modified Version, သို့မဟုတ် မည်သည့် Derivative Work ကိုမဆို တည်ဆောက်၊ ပြုပြင်၊ အသုံးပြု၊ အိမ်ရှင်အဖြစ်ဆောင်ရွက်၊ စိတ်ကြိုက်ပြုပြင်၊ fork, port, အမည်ပြောင်၊ သို့မဟုတ် အခြားနည်းဖြင့် အလုပ်လုပ်ရန် စေ့စပ်၊ ငှားရမ်း၊ စာချုပ်ချုပ်၊ ညွှန်ကြား၊ သို့မဟုတ် အခြားနည်းဖြင့် အသုံးပြုသော မည်သည့်အဖွဲ့အစည်းမဆို ထိုကဲ့သို့သော Third-Party Developer နှင့် ခွင့်ပြုချက်မရှိသော စီးပွားဖြစ်အသုံးပြုမှု၊ ဤလိုင်စင်၏ ချိုးဖောက်မှု၊ သို့မဟုတ် လိုင်စင်ရှင်၏ ဉာဏပစ္စည်းမူပိုင်ခွင့်ချိုးဖောက်မှုအတွက် ထိုကဲ့သို့သော Third-Party Developer နှင့် ပူးတွဲနှင့် တစ်ဦးချင်း တာဝန်ယူရမည်။**

အထက်ပါအချက်၏ ယေဘုယျသဘောကိ် ကန့်သတ်ခြင်းမပြုဘဲ:

1. **ပူးတွဲနှင့် တစ်ဦးချင်း တာဝန်ယူမှု (Joint and Several Liability)** ။ အလုပ်ကို အကျိုးခံစားခွင့်ရှိသူ အဖွဲ့အစည်း (contracting company) နှင့် ၎င်းမှ စေ့စပ်ထားသော Third-Party Developer တစ်ဦးစီ (နှင့် ထို Third-Party Developer ၏ ကန်ထရိုက်ခွဲ (subcontractor) တစ်ဦးစီ) သည် လိုင်စင်ရှင်ထံပေးဆောင်ရမည့် ပျက်စီးဆုံးရှုံးမှု၊ ပျက်စီးဆုံးရှုံးမှုကြေး၊ လိုင်စင်ကြေးများ၊ လွှဲပြောင်းပေးအပ်ခြင်း၊ နှင့် ကုန်ကျစရိတ်အားလုံး၏ တစ်ခုလုံးအတွက် တစ်ဦးချင်းစီနှင့် အတူတူ အပြည့်အဝ တာဝန်ယူရမည်။ လိုင်စင်ရှင်သည် ၎င်းတို့ထဲမှ တစ်ဦး သို့မဟုတ် တစ်ဦးထက်ပိုသော မည်သည့်ပေါင်းစပ်မှုကိုမဆို ပမာဏအပြည့်အစုအတွက် တရားစွဲနိုင်သည်။

2. **လွှဲအပ်ခြင်း၏ ကာကွယ်မှုမရှိပါ (No Defense of Delegation)** ။ စာချုပ်ချုပ်သောကုမ္ပဏီသည် Third-Party Developer က တစ်ဦးတည်းသာ တာဝန်ရှိသည်ဟု ဆိုခြင်း၊ စာချုပ်ချုပ်သောကုမ္ပဏီက ကုဒ်ကို မရေးထားခြင်း၊ Third-Party Developer က အလုပ်အား လိုင်စင်ရသည်ဟု ကိုယ်စားပြုထားခြင်း၊ သို့မဟုတ် စာချုပ်ချုပ်သောကုမ္ပဏီက Software သည် Fincept Terminal (သို့) Derivative Work တစ်ခုဖြစ်ကြောင်း မသိခဲ့ခြင်းတို့ဖြင့် တာဝန်မှ လွတ်မြောက်နိုင်မည် မဟုတ်ပါ။ လိုင်စင်လိုက်နာမှုကို အတည်ပြုရန် တာဝန်သည် **လွှဲအပ်၍မရသော (non-delegable)** ဖြစ်သည်။

3. **အသိပညာမရှိခြင်း၏ ကာကွယ်မှုမရှိပါ (No Defense of Lack of Knowledge)** ။ ဤလိုင်စင်၊ LICENSE ဖိုင်၊ README နှင့် Software တွင် ထည့်သွင်းထားသော လိုင်စင်အသိပေးချက်များ၏ အများသူငှာ ရရှိနိုင်မှုသည် အသိအမှတ်ပြုချက် (constructive notice) ဖြစ်သည်။ လိုင်စင်သတ်မှတ်ချက်များကို မသိခြင်းသည် စာချုပ်ချုပ်သောကုမ္ပဏီ သို့မဟုတ် Third-Party Developer အတွက် ကာကွယ်မှုမဟုတ်ပါ။

4. **တွဲဖက်ကုမ္ပဏီများ၊ ဆက်ခံသူများ၊ ဖီးနစ်အဖွဲ့အစည်းများ (Phoenix entities)၊ နှင့် အကျိုးခံစားရသူများ (Beneficial Users)** ။ တာဝန်ယူမှုသည် မိဘကုမ္ပဏီများ၊ လက်အောက်ကုမ္ပဏီများ၊ တွဲဖက်ကုမ္ပဏီများ၊ ဆက်ခံသူများ၊ တာဝန်လွှဲပြောင်းခံရသူများ (assignees)၊ ဖက်စပ်လုပ်ဖော်ကိုင်ဖက်များ၊ ဝယ်ယူသူများ (acquirers)၊ နှင့် ကော်ပိုရိတ်ပုံစံ၊ စာချုပ်အဆက်အစပ်၊ သို့မဟုတ် ၎င်းမရှိခြင်းကိုမခွဲခြားဘဲ Software ကို အသုံးပြုခြင်း၏ အကျိုးခံစားခွင့်ကို **ရန်ပုံငွေ (funds)၊ ထိန်းချုပ် (controls)၊ အကျိုးခံစား (benefits from)၊ ဝန်ထမ်းများထောက်ပံ့ (supplies personnel to)၊ အခြေခံအဆောက်အအုံထောက်ပံ့၊ ဒေတာထောက်ပံ့၊ သို့မဟုတ် Software အသုံးပြုခြင်းဖြင့် ထုတ်လုပ်သော output, derivative data, သို့မဟုတ် services များကို လက်ခံရရှိ** သော မည်သည့်ပုဂ္ဂိုလ် သို့မဟုတ် အဖွဲ့အစည်းအထိ တိုးချဲ့သည်။ **တရားဝင် စေ့စပ်မှု (formal engagement) မဟုတ်ဘဲ အကျိုးခံစားမှု (beneficial use) သည် တာဝန်ယူမှုအတွက် စမ်းသပ်မှု (test) ဖြစ်သည်။** ကော်ပိုရိတ် ပြန်လည်ဖွဲ့စည်းခြင်း၊ ပိုင်ဆိုင်မှုလွှဲပြောင်းခြင်း၊ အမည်ပြောင်းလဲခြင်း၊ ဖျက်သိမ်းခြင်း၊ ပေါင်းစည်းခြင်း၊ ဝယ်ယူခြင်း၊ အစုရှယ်ယာရောင်းချခြင်း၊ သို့မဟုတ် ပြန်လည်ပေါင်းစည်းခြင်းတို့သည် တာဝန်ယူမှုကို မပယ်ဖျက်ပါ။ ချိုးဖောက်မှုမဖြစ်မီ သို့မဟုတ် ပြီးနောက် နှစ်ဆယ့်လေး (၂၄) လအတွင်း၊ ဝန်ထမ်းများ၏ နှစ်ဆယ့်ငါးရာခိုင်နှုန်း (၂၅%) ထက်ပိုသော တူညီသော ဝန်ထမ်းများကို ခန့်ထား၊ တူညီသောနေရာကို သိမ်းပိုက်၊ တူညီသောဖောက်သည်များကို ဝန်ဆောင်မှုပေး၊ သို့မဟုတ် သိသိသာသာ တူညီသော code base မှ လုပ်ကိုင်သော မည်သည့်အဖွဲ့အစည်းမဆို ကော်ပိုရိတ်ပုံစံ သို့မဟုတ် ပိုင်ဆိုင်မှုလွှဲပြောင်းမှုကို မခွဲခြားဘဲ ဤလိုင်စင်၏ ရည်ရွယ်ချက်များအားလုံးအတွက် ဆက်ခံသူအဖြစ် သတ်မှတ်ခံရမည်။ တာဝန်ယူမှုသည် ဝန်ထမ်းအေဂျင်စီများ၊ BPO/KPO ကုမ္ပဏီများ၊ ကန်ထရိုက်တာများ၊ ကန်ထရိုက်ခွဲများ၊ အလုပ်သမားငှားရမ်းသည့် အဖွဲ့အစည်းများ (labor-leasing entities)၊ သို့မဟုတ် ကြားခံအဖွဲ့အစည်းများ၏ ကွင်းဆက်မှတစ်ဆင့်၊ ကြားခံအရေအတွက် မည်မျှပင်ရှိစေကာမူ တိုးချဲ့သည်။

   **ဝယ်ယူသူ၏ တာဝန်ယူမှု (Acquirer Liability)** ။ Software ကို အသုံးပြုခဲ့သော မည်သည့်အဖွဲ့အစည်းကိုမဆို ဝယ်ယူသူသည် ဝယ်ယူခြင်းမတိုင်မီ၊ အတွင်းနှင့် ပြီးနောက် ဖြစ်ပွားသော ခွင့်ပြုချက်မရှိသော အသုံးပြုမှုအတွက် ပူးတွဲနှင့် တစ်ဦးချင်း တာဝန်ယူရမည်။ အရောင်းအ၀ယ်ကို ပိုင်ဆိုင်မှုဝယ်ယူခြင်း (asset purchase)၊ အစုရှယ်ယာဝယ်ယူခြင်း (stock purchase)၊ ပေါင်းစည်းခြင်း (merger)၊ သို့မဟုတ် အခြားပုံစံဖြင့် ဖွဲ့စည်းသည်ဖြစ်စေ၊ နှင့် ဝယ်ယူမှုသဘောတူညီချက်တွင် မည်သည့် တာဝန်မယူကြောင်း အပိုဒ် (non-assumption clause) မဆို ခွဲခြားမှုမရှိပါ။ ဝယ်ယူသူများသည် ဤလိုင်စင်အား လုပ်ဆောင်ခြင်းဖြင့် Software-လိုင်စင်ဆိုင်ရာ တာဝန်များကို လက်ခံသည်ဟု သတ်မှတ်ပြီး၊ ဤစာတမ်း၏ အများသူငှာ ရရှိနိုင်မှုဖြင့် အကြောင်းကြားခြင်း ဖြစ်သည်။ လိုင်စင်ရသူသည် ငွေကြေးဆိုင်ရာ လုပ်ဆောင်နိုင်စွမ်း (finance functionality) ပါရှိသော မည်သည့် တတိယပုဂ္ဂိုလ် ဆော့ဖ်ဝဲကိုမဆို အသုံးမပြုမီ ဆော့ဖ်ဝဲပါဝင်မှု ပိုင်းခြားစိတ်ဖြာမှု (software-composition analysis — SCA) ကို ပြုလုပ်ရန် သို့မဟုတ် လုပ်ဆောင်စေရမည်။ သို့မဟုတ်ပါက ရည်ရွယ်ချက်ရှိရှိ မျက်ကွယ်ပြုခြင်း (willful blindness) ဖြစ်သည်။

   **လွှဲပြောင်းခြင်းအတွက် ရပ်နားခြင်း (Standstill on Transfer)** ။ ရပ်တန့်ရန် သတိပေးစာ (cease-and-desist notice)၊ စစ်ဆေးရန် တောင်းဆိုချက် (audit demand)၊ သို့မဟုတ် ချိုးဖောက်မှုသတိပေးချက် (notice of breach) ကို လက်ခံရရှိပြီးနောက်၊ လိုင်စင်ရသူသည် လိုင်စင်ရှင်၏ ကြိုတင်လိပ်မူဖြင့် ခွင့်ပြုချက်မရှိဘဲ Software၊ မည်သည့် Derivative Work၊ ဆက်စပ်ဝန်ထမ်းများ၊ သို့မဟုတ် ဆက်စပ်အခြေခံအဆောက်အအုံကို မည်သည့် တွဲဖက်ကုမ္ပဏီ သို့မဟုတ် တတိယပုဂ္ဂိုလ်ထံမဆို လွှဲပြောင်း၊ လိုင်စင်ချပေး၊ တာဝန်လွှဲပြောင်း၊ သို့မဟုတ် လိုင်စင်ခွဲပေးခြင်း မပြုရ။ ထိုသို့သော လွှဲပြောင်းမှုများသည် လိုင်စင်ရှင်အား ဆန့်ကျင်သည့်အနေဖြင့် ပျက်ပြယ်သည် (void)။

5. **အရာရှိများ၊ ဒါရိုက်တာများ၊ နှင့် ပိုင်ရှင်များ (Officers, Directors, and Principals)** ။ သက်ဆိုင်ရာဥပဒေက ခွင့်ပြုသည့်နေရာတွင်၊ လိုင်စင်ရှင်သည် ချိုးဖောက်နေသော အဖွဲ့အစည်းတစ်ခု၏ အရာရှိများ၊ ဒါရိုက်တာများ၊ မိတ်ဖက်များ၊ နှင့် ထိန်းချုပ်သည့် အစုရှယ်ယာရှင်များကို ရည်ရွယ်ချက်ရှိရှိ ချိုးဖောက်မှု (willful infringement)၊ ချိုးဖောက်မှုနှင့် ဆက်စပ်၍ သစ္စာခံတာဝန် (fiduciary duty) ချိုးဖောက်မှု၊ သို့မဟုတ် လိမ်လည် ဖုံးကွယ်ခြင်း (fraudulent concealment) တို့အတွက် တရားစွဲဆိုမှုများ ပြုလုပ်နိုင်သည်။

6. **စာချုပ်ချုပ်သောကုမ္ပဏီမှ တာဝန်စစ်ပေးခြင်း (Indemnification by the Contracting Company)** ။ စာချုပ်ချုပ်သောကုမ္ပဏီသည် ၎င်းမှ စေ့စပ်ထားသော မည်သည့် Third-Party Developer ၏ လုပ်ရပ် သို့မဟုတ် ပျက်ကွက်မှုများမှ ပေါ်ပေါက်လာသော တောင်းဆိုမှု၊ ဆုံးရှုံးမှု၊ ပျက်စီးဆုံးရှုံးမှု၊ အခကြေးငွေ၊ နှင့် ကုန်ကျစရိတ်အားလုံးမှ လိုင်စင်ရှင်အား တာဝန်စစ်ပေးရန် (indemnify)၊ ကာကွယ်ရန် (defend)၊ နှင့် အန္တရာယ်မှ ကင်းရှင်းစေရန် (hold harmless) တာဝန်ရှိသည်။ စာချုပ်ချုပ်သောကုမ္ပဏီက ထိုသို့သော လုပ်ရပ်များကို ညွှန်ကြား၊ သိရှိ၊ သို့မဟုတ် သဘောတူသည်ဖြစ်စေ မခွဲခြားဘဲ ဤတာဝန်သက်ရောက်သည်။

7. **တရားစွဲခံရခြင်းတွင် ပူးပေါင်းစေရန် အခွင့်အရေး (Right to Joinder)** ။ Third-Party Developer တစ်ဦးအား တရားစွဲသည့် အမှုတွင်၊ လိုင်စင်ရှင်သည် ၎င်း၏ တစ်ဦးတည်းသော ဆန္ဒအလျောက် စာချုပ်ချုပ်သောကုမ္ပဏီကို (နှင့် အပြန်အလှန်လည်း) တရားခံတွဲအဖြစ် တူညီသော တရားစွဲဆိုမှုတွင် ပါဝင်စေနိုင်ပြီး၊ ပါတီများသည် ထိုသို့သော ပူးပေါင်းခြင်းကို ကြိုတင်၍ သဘောတူညီပါသည်။

8. **အများသူငှာ အသိပေးရန် တာဝန် (Public Notice Obligation)** ။ လိုင်စင်ရှင်၏ တောင်းဆိုမှုအရ၊ Software နှင့် အလုပ်လုပ်ရန် Third-Party Developer တစ်ဦးကို စေ့စပ်သော စာချုပ်ချုပ်သောကုမ္ပဏီသည် လုပ်ငန်းရက် ဆယ်ရက် (၁၀) အတွင်း အောက်ပါတို့ကို လိုင်စင်ရှင်အား ပေးအပ်ရမည်- (က) Third-Party Developer ၏ ဥပဒေအမည်နှင့် ဆက်သွယ်ရန်အသေးစိတ်၊ (ခ) လုပ်ဆောင်မှု၏ နယ်ပယ်၊ (ဂ) စီးပွားရေးဆိုင်ရာ စျေးနှုန်းသတ်မှတ်ချက်များအထိသာ အနီရောင်ဖြတ်ထားသော (redacted) သက်ဆိုင်ရာ စာချုပ်ချုပ် သို့မဟုတ် လုပ်ငန်းနယ်ပယ် (statement of work) ၏ မိတ္တူများ၊ နှင့် (ဃ) Software သို့ ဝင်ရောက်ခွင့်ရှိသော repositories များ၊ အသုံးပြုမှုနေရာများ၊ နှင့် ဝန်ထမ်းများ၏စာရင်း။

9. **တာဝန်ယူမှုမှ ကန်ထရိုက်ခွဲမခြင်း (No Subcontracting Out of Liability)** ။ စာချုပ်ချုပ်သောကုမ္ပဏီနှင့် Third-Party Developer အကြား လိုင်စင်ရှင်အား တာဝန်ပေးအပ် သို့မဟုတ် စွန့်လွှတ်ရန် ကြိုးပမ်းသော မည်သည့် စာချုပ်ဆိုင်ရာ ပြဌာန်းချက် — ၎င်းတို့ကြား လည်ပတ်သော တာဝန်စစ်ပေးခြင်း (indemnities)၊ အန္တရာယ်ကင်းရှင်းရေး သဘောတူညီချက်များ (hold-harmless agreements)၊ သို့မဟုတ် "ရှိသလိုး" ငြင်းဆိုချက်များ (as-is disclaimers) အပါအဝင် — သည် ထိုသဘောတူညီချက်၏ ပါတီမဟုတ်သော လိုင်စင်ရှင်အား ဆန့်ကျင်သည့်အနေဖြင့် အကျိုးသက်ရောက်မှု မရှိပါ။

ဤပုဒ်မ 14 သည် ဤလိုင်စင်၏ **အဓိက သတ်မှတ်ချက် (material term)** တစ်ခုဖြစ်သည်။ ၎င်းကို မည်သည့်ချိုးဖောက်မှုမဆို ရပ်စဲခြင်း၊ တားမြစ်မိန့် (injunctive relief)၊ နှင့် ပုဒ်မ 13 တွင်ဖော်ပြထားသော အများဆုံး ပျက်စီးဆုံးရှုံးမှုကြေးများ အတွက် ချက်ချင်း အကြောင်းရင်း (immediate grounds) ဖြစ်သည်။

---

## ၁၅။ တာဝန်ယူမှု ကန့်သတ်ချက် (Limitation of Liability)

သက်ဆိုင်ရာဥပဒေက ခွင့်ပြုသော အများဆုံးအတိုင်းအတာအထိ၊ ဤလိုင်စင်အောက် သို့မဟုတ် ဤလိုင်စင်နှင့် ဆက်စပ်၍ လိုင်စင်ရှင်၏ စုစုပေါင်း တာဝန်ယူမှုသည် တာဝန်ဖြစ်ပေါ်စေသော အဖြစ်အပျက်မဖြစ်မီ ဆယ့်နှစ်လ (၁၂) လအတွင်း လိုင်စင်ရသူမှ လိုင်စင်ရှင်ထံ ပေးဆောင်ခဲ့သော စုစုပေါင်းအခကြေးငွေ၊ သို့မဟုတ် အိန္ဒိယရူပီးတစ်ရာ (₹၁၀၀) တို့တွင် မည်သည့်ပမာဏက ပိုကြီးလဲ ထိုပမာဏထက် မကျော်လွန်စေရ။ လိုင်စင်ရှင်သည် သွယ်ဝိုက်သော၊ အကျိုးဆက်ဖြစ်သော၊ မတော်တဆဖြစ်သော၊ အထူးသဖြင့်၊ သာဓကပြသော၊ သို့မဟုတ် အပြစ်ပေးသော ပျက်စီးဆုံးရှုံးမှုများ၊ ဆုံးရှုံးသော အမြတ်ငွေ၊ ပျောက်ဆုံးသော ဒေတာ၊ ဂုဏ်သတင်းပျက်စီးမှု၊ သို့မဟုတ် အရောင်းအဝယ် ဆုံးရှုံးမှုများအတွက် ထိုကဲ့သို့သော ပျက်စီးဆုံးရှုံးမှုများ ဖြစ်နိုင်ခြေကို အကြံပေးထားသည့်တိုင် မည်သည့်အခါမျှ တာဝန်ယူမည် မဟုတ်ပါ။ ဤကန့်သတ်ချက်များသည် စာချုပ်၊ မတရားမှု (tort)၊ တင်းကျပ်သော တာဝန်ယူမှု (strict liability)၊ သို့မဟုတ် အခြားနည်းဖြင့် အရေးယူသည့် ပုံစံကို မခွဲခြားဘဲ သက်ရောက်သည်။

## ၁၆။ အုပ်ချုပ်သည့်ဥပဒေနှင့် တရားစီရင်ပိုင်ခွင့် (Governing Law and Jurisdiction)

ဤလိုင်စင်ကို အိန္ဒိယနိုင်ငံတော်သမ္မတနိုင်ငံ (Republic of India) ၏ ဥပဒေများဖြင့် အုပ်ချုပ်ပြီး အဓိပ္ပာယ်ဖွင့်ဆိုရမည်။ အဆိုပါ ဥပဒေများ၏ ပဋိပက္ခဖြစ်ရပ်ဆိုင်ရာ အခြေခံမူများကို ထည့်သွင်းစဉ်းစားခြင်း မရှိစေရ။ ကုန်ပစ္စည်းများ နိုင်ငံတကာ ရောင်းချမှုဆိုင်ရာ ကုလသမဂ္ဂ သဘောတူစာချုပ် (United Nations Convention on Contracts for the International Sale of Goods) အကျုံးမဝင်ပါ။

**အဓိက တရားရုံး (Primary Forum)** ။ **အိန္ဒိယနိုင်ငံ၊ ဒေလီ (Delhi)** ရှိ တရားရုံးများသည် လိုင်စင်ရှင်၏ ရွေးချယ်မှုအရ ဤလိုင်စင်မှ ပေါ်ပေါက်လာသော သို့မဟုတ် ဆက်စပ်သော အငြင်းပွားမှုများအားလုံးကို အထူးတရားစီရင်ပိုင်ခွင့် (exclusive jurisdiction) ရှိသည်။ လိုင်စင်ရသူသည် ထိုကဲ့သို့သော တရားရုံးများ၏ ပုဂ္ဂိုလ်ဆိုင်ရာ တရားစီရင်ပိုင်ခွင့် (personal jurisdiction) ကို နောက်မဆုတ်နိုင်အောင် (irrevocably) လက်အောက်ခံပြီး လိုင်စင်ရှင်မှ တင်သွင်းသော တရားစွဲဆိုမှုများတွင် **အဆင်မပြေသော ဖိုရမ်အခြေပြု ကာကွယ်မှု (forum-non-conveniens defense)**၊ အဆင်မပြေသော ဖိုရမ်အတွက် ကန့်ကွက်ချက်၊ နှင့် ပုဂ္ဂိုလ်ဆိုင်ရာ တရားစီရင်ပိုင်ခွင့် မရှိခြင်းအပေါ် အခြေပြု ကန့်ကွက်ချက်တို့ကို **ရှင်းလင်းစွာ စွန့်လွှတ်သည်**။

**ပူးတွဲ ဖိုရမ်များ (Concurrent Forums — လိုင်စင်ရှင်၏ ရွေးချယ်မှုအရ)** ။ ဒေလီအပြင်၊ လိုင်စင်ရှင်သည် ၎င်း၏ တစ်ဦးတည်းသော ဆန္ဒအရ လိုင်စင်ရသူနေထိုင်သော၊ ပေါင်းစည်းထားသော၊ ပိုင်ဆိုင်မှုရှိသော၊ Software ကိုလုပ်ဆောင်သော၊ သို့မဟုတ် Software မှဆင်းသက်လာသော ဝန်ဆောင်မှုများကို ညွှန်ကြားသော မည်သည့် ဖိုရမ်တွင်မဆို တရားစွဲဆိုမှုများ ဆောင်နိုင်သည်။ လိုင်စင်ရသူသည် ထိုကဲ့သို့သော ဖို�ရမ်တစ်ခုခုတွင် သီးသန့်မဟုတ်သော တရားစီရင်ပိုင်ခွင့် (non-exclusive jurisdiction) ကို ကြိုတင်သဘောတူသည်။ ဒေသဆိုင်ရာဥပဒေက ခွင့်ပြုပါက၊ လိုင်စင်ရသူသည် Software မှဆင်းသက်လာသော ဝန်ဆောင်မှုများကို လုပ်ဆောင်၊ ထိန်းသိမ်း၊ သို့မဟုတ် ညွှန်ကြားသည့် မည်သည့် တရားစီရင်ပိုင်ခွင့်တွင်မဆို တရားစွဲဆိုမှုဆိုင်ရာ စာရွက်စာတမ်းများ (service of process) အတွက် လိုင်စင်ရှင်အား ၎င်း၏ ကိုယ်စားလှယ် (agent) အဖြစ် နောက်မဆုတ်နိုင်အောင် ခန့်အပ်သည်။

**ကန့်သတ်ကာလ ရည်ရွယ်ချက်အတွက် စဉ်ဆက်မပြတ် ချိုးဖောက်မှု (Continuous Breach for Limitation Purposes)** ။ Software ကို ခွင့်ပြုချက်မရှိဘဲ လုပ်ဆောင်ခြင်း၊ ဖြန့်ဝေခြင်း၊ ပြုပြင်ခြင်း၊ သို့မဟုတ် အသုံးပြုခြင်းတစ်ခုစီနှင့် ဆက်လက်အသုံးပြုခြင်း၏ တစ်ရက်စီသည် ကန့်သတ်ကာလ (limitation period) ရည်ရွယ်ချက်များအတွက် **သီးခြား သီးသန့်ဖြစ်ပေါ်သော ချိုးဖောက်မှု (separately accruing breach)** တစ်ခုဖြစ်သည်။ လိုင်စင်ကြေးများ၊ ပျက်စီးဆုံးရှုံးမှုကြေးများ၊ အမြတ်အစွန်း လွှဲပြောင်းပေးအပ်ခြင်းနှင့် အခြားသော ငွေကြေးဆိုင်ရာ ကုစားနည်းများသည် စဉ်ဆက်မပြတ် ချိုးဖောက်မှု အခြေခံဖြင့် ပေါ်ပေါက်သည်။ ဖြစ်ပေါ်လာသော ချိုးဖောက်မှုတစ်ခုစီအတွက် ကန့်သတ်ကာလသည် ထိုချိုးဖောက်မှု၏ ရက်စွဲမှ လွတ်လပ်စွာ စတင်သည်။

**လွှတ်ငြိမ်းခြင်း၊ တားဆီးခြင်း၊ သို့မဟုတ် ဆိတ်ဆိတ်နေခြင်း မရှိပါ (No Waiver, Estoppel, or Acquiescence)** ။ လိုင်စင်ရှင်၏ ဆိတ်ဆိတ်နေခြင်း၊ လုပ်ဆောင်မှုမရှိခြင်း၊ နှောင့်နှေးခြင်း၊ အများသူငှာ ဆက်သွယ်ရေး၊ လူမှုမီဒီယာ ပို့စ်၊ စျေးကွက်ရှာဖွေရေးကြေညာချက်၊ ကွန်ဖရင့်ဟောပြောချက်၊ သို့မဟုတ် အပြန်အလှန်ဆက်ဆံမှုပုံစံတို့သည် လွှတ်ငြိမ်းခြင်း (waiver)၊ လိုင်စင်၊ တားဆီးခြင်း (estoppel)၊ ဆိတ်ဆိတ်သဘောတူခြင်း (acquiescence)၊ လျစ်လျူရှုခြင်းကြောင့် သတ်မှတ်ကာလအတွင်း အခွင့်အရေးဆုံးရှုံးခြင်း (laches)၊ သို့မဟုတ် သွယ်ဝိုက်သောလိုင်စင် (implied license) ဖြစ်စေမည် မဟုတ်ပါ။ Software ကို "အခမဲ့" သို့မဟုတ် "open source" အဖြစ်ဖော်ပြသော လူသိရှင်ကြား စျေးကွက်ရှာဖွေရေး သို့မဟုတ် မီဒီယာသည် တစ်ဦးချင်း စီးပွားဖြစ်မဟုတ်သော အသုံးပြုသူများအတွက် AGPL-3.0 ရွေးချယ်မှုကိုသာ ရည်ညွှန်းပြီး စီးပွားဖြစ်အသုံးပြုမှုအတွက် လိုင်စင်၊ သွယ်ဝိုက်သောလိုင်စင်၊ လွှတ်ငြိမ်းခြင်း၊ သို့မဟုတ် ကိုယ်စားပြုချက်တစ်ခုအဖြစ် အဓိပ္ပာယ်ဖွင့်ဆိုခြင်း မရှိရ။ လိုင်စင်ရသူသည် မည်သည့်ပြင်ပအဖွဲ့အစည်းကိုမဆို ဆန့်ကျင်ပြီး လိုင်စင်ရှင်၏ အတည်ဖြစ်စေခြင်း ပျက်ကွက်ခြင်းကို အားမကိုးရပါ။

**ဥပဒေအရ ထွင်းထုခြင်းများ (Statutory Carve-Outs) ကို ကျဉ်းမြောင်းစွာ အဓိပ္ပာယ်ဖွင့်ဆိုခြင်း** ။ လိုင်စင်ရသူမှ ဆိုကိုင်းသည့် မည်သည့် ဥပဒေအရ မျှတသော အရောင်းအဝယ် (fair dealing)၊ တရားဝင်အသုံးပြုမှု (fair use)၊ အပြန်အလှန် လုပ်ဆောင်နိုင်မှုအတွက် နောက်ပြန်ဖော်ထုတ်ခြင်း (reverse-engineering-for-interoperability)၊ သို့မဟုတ် အစိုးရအသုံးပြုမှု (government-use) အခွင့်အရေးကို သက်ဆိုင်ရာ ဥပဒေ၏ လိုအပ်သော အနိမ့်ဆုံး အတိုင်းအတာဖြင့်သာ အဓိပ္ပာယ်ဖွင့်ဆိုရမည်။ အသုံးပြုမှုသည် ဥပဒေအရ ထွင်းထုခြင်းအတွင်း ကျရောက်ကြောင်း သက်သေပြရန် တာဝန်သည် လိုင်စင်ရသူတွင် ရှိသည်။ ဥပဒေအရ အနိမ့်ဆုံးထက် ကျော်လွန်သော မည်သည့်အသုံးပြုမှုမဆို ခွင့်ပြုချက်မရှိပါ။ မည်သည့်အစိုးရ၊ အချုပ်အခြာအာဏာပိုင်၊ သို့မဟုတ် ပြည်သူ့ကဏ္ဍမှ အဖွဲ့အစည်းတစ်ခုခုက ဥပဒေအရ ထွင်းထုခြင်းကို ဆိုကိုင်းပါက၊ စီးပွားဖြစ်လိုင်စင်ကြေးများကို အစိုးရအသုံးပြုသူများအတွက် ထုတ်ဝေထားသောနှုန်းဖြင့် ပေးချေရမည်ဖြစ်ပြီး၊ စစ်ဆေးခြင်း၊ ရည်ညွှန်းခြင်း၊ သို့မဟုတ် ကုန်အမှတ်တံဆိပ်အခွင့်အရေးများကို လွှတ်ငြိမ်းခြင်း မရှိပါ။

**ကမ္ဘာတစ်ဝှမ်း တရားမျှတမှုဆိုင်ရာ ကုစားနည်း (Worldwide Equitable Relief)** ။ ဤပုဒ်မ၏ မည်သည့်ပြဌာန်းချက်ကိုမဆို မထိခိုက်စေဘဲ၊ ၎င်း၏ ဉာဏပစ္စည်းမူပိုင်ခွင့်ကို ကာကွယ်ရန်အတွက် လိုင်စင်ရှင်သည် ကမ္ဘာပေါ်ရှိ သင့်လျော်သော တရားစီရင်ပိုင်ခွင့်ရှိ မည်သည့်တရားရုံးတွင်မဆို တားမြစ်မိန့် သို့မဟုတ် တရားမျှတမှုဆိုင်ရာ ကုစားနည်း (equitable relief) ကို ရှာဖွေနိုင်သည်။

## ၁၇။ အထွေထွေ ပြဌာန်းချက်များ

**ခွဲခြား၍ရနိုင်မှု (Severability)** ။ ဤလိုင်စင်၏ မည်သည့်ပြဌာန်းချက်ကိုမဆို မမှန်ကန်ကြောင်း သို့မဟုတ် အတည်မဖြစ်နိုင်ကြောင်း တွေ့ရှိပါက၊ ကျန်ရှိသော ပြဌာန်းချက်များသည် အပြည့်အဝ အင်အားရှိနေမည်ဖြစ်ပြီး၊ မမှန်ကန်သော ပြဌာန်းချက်ကို ၎င်း၏ ရည်ရွယ်ချက်ကို ထိန်းသိမ်းထားချိန်တွင် အတည်ဖြစ်စေရန် လိုအပ်သော အနည်းဆုံးအတိုင်းအတာအထိ ပြုပြင်ရမည်။

**လုံးလုံးသဘောတူညီချက် (Entire Agreement)** ။ ဤလိုင်စင်သည် အကောင်အထည်ဖော်သော မည်သည့် Order Form နှင့်အတူ ပါတီများကြား လုံးလုံးသဘောတူညီချက် ဖြစ်ပြီး အကြောင်းအရာနှင့်ပတ်သက်၍ ယခင် သို့မဟုတ် ခေတ်ပြိုင် ပြောဆိုမှု၊ စာဖြင့်ဖြစ်စေ၊ နှုတ်ဖြင့်ဖြစ်စေ အားလုံးကို အစားထိုးသည်။

**လွှတ်ငြိမ်းခြင်း မရှိပါ (No Waiver)** ။ လိုင်စင်ရှင်မှ မည်သည့်အခွင့်အရေးကိုမဆို ကျင့်သုံးရန် ပျက်ကွက်ခြင်း သို့မဟုတ် နှောင့်နှေးခြင်းသည် လွှတ်ငြိမ်းခြင်းအဖြစ် အလုပ်မလုပ်နိုင်၊ တစ်စိတ်တစ်ပိုင်း ကျင့်သုံးခြင်းသည် အခြားသော နောက်ထပ် ကျင့်သုံးမှုကိုဖြစ်စေ၊ နောက်ထပ် ကျင့်သုံးမှုကို တားဆီးနိုင်မည် မဟုတ်ပါ။

**တာဝန်လွှဲပြောင်းခြင်း (Assignment)** ။ လိုင်စင်ရသူသည် လိုင်စင်ရှင်၏ ကြိုတင်လိပ်မူဖြင့် ခွင့်ပြုချက်မရှိဘဲ၊ ဥပဒေအရဖြစ်စေ၊ အခြားနည်းဖြင့်ဖြစ်စေ ဤလိုင်စင်ကို လွှဲပြောင်း သို့မဟုတ် အပ်နှင်းခြင်း မပြုရ။ ဤပြဌာန်းချက်ကို ချိုးဖောက်၍ ပြုလုပ်သော မည်သည့် ကြိုးပမ်းမှုမဆို ပျက်ပြယ်သည် (void)။ လိုင်စင်ရှင်သည် ဤလိုင်စင်ကို လွတ်လပ်စွာ လွှဲပြောင်းနိုင်သည်။

**အင်အားသာလွန်မှု (Force Majeure)** ။ မည်သည့်ပါတီမျှ ၎င်း၏ ကျိုးကြောင်းဆီလျော်သော ထိန်းချုပ်မှုထက်ကျော်လွန်သော အကြောင်းများကြောင့် လုပ်ဆောင်ရန် ပျက်ကွက်ခြင်းအတွက် တာဝန်မယူရ၊ သို့သော် ငွေပေးချေရန် တာဝန်ဝတ္တရားများ မပါဝင်ပါ။

**အကြောင်းကြားစာများ (Notices)** ။ လိုင်စင်ရှင်ထံ အကြောင်းကြားစာများကို support@fincept.in သို့ ပေးပို့ရမည်ဖြစ်ပြီး၊ Fincept Corporation ၏ မှတ်ပုံတင်ထားသောရုံးသို့ စာရင်းသွင်းစာတိုက်ဖြင့် မိတ္တူတစ်စောင်ထပ်ပို့ရမည်။

**တင်ပို့မှုထိန်းချုပ်ရေး (Export Controls)** ။ လိုင်စင်ရသူသည် အိန္ဒိယ၊ အမေရိကန်၊ ဥရောပသမဂ္ဂ၊ နှင့် အခြားသက်ဆိုင်ရာ တရားစီရင်ပိုင်ခွင့်အားလုံး၏ အကျုံးဝင်သော တင်ပို့မှုထိန်းချုပ်ရေးဥပဒေများကို လိုက်နာရမည်။

**ပြန်လည်လွှဲပြောင်းခြင်း မပြုနိုင် (Non-Transferability)** ။ လိုင်စင်များသည် ပြန်လည်လွှဲပြောင်းခြင်း မပြုနိုင်ပါ။ နှစ်စဉ် လိုင်စင်များသည် သက်တမ်းတိုးရက်မတိုင်မီ ရက်ပေါင်း ၃၀ အတွင်း အကြောင်းကြားချက်ဖြင့် မပယ်ဖျက်ပါက နှစ်စဉ် သက်တမ်းတိုးသည်။

## ၁၈။ စီးပွားဖြစ်လိုင်စင် ရယူနည်း

1. **ဆက်သွယ်ပါ** — အောက်ပါတို့ဖြင့် **support@fincept.in** သို့ အီးမေးလ်ပေးပို့ပါ: (က) အဖွဲ့အစည်း၏ ဥပဒေအမည်၊ (ခ) ပေါင်းစည်းထားသော နိုင်ငံ၊ (ဂ) အသုံးပြုရန် ရည်ရွယ်ထားသော အခြေအနေ၊ (ဃ) အသုံးပြုသူအရေအတွက်၊ (င) သင်သည် Modified Version သို့မဟုတ် Derivative Work ကို အသုံးပြုနေသည် သို့မဟုတ် အသုံးပြုရန် စီစဉ်သည် ရှိ/မရှိ။
2. **သဘောတူညီချက်** — စီးပွားဖြစ်လိုင်စင်သဘောတူညီချက် (ဤစာတမ်းကို Order Form အဖြစ် တရားဝင်ဖြစ်စေခြင်း) ကို ပြန်လည်သုံးသပ်ပြီး လက်မှတ်ရေးထိုးပါ။
3. **ငွေပေးချေခြင်း** — Order Form နှင့်အညီ ငွေပေးချေမှုကို ပြီးပါစေ။
4. **ဝင်ရောက်ခွင့်** — အထောက်အထားများ (credentials) နှင့် စီးပွားဖြစ်အသုံးပြုခွင့်အခွင့်အရေးများကို လက်ခံရရှိပါ။
